Capítulo 2. Manipulando pixels em uma imagem

O objetivo dessa lição é mostrar como manipular os pixels de uma imagem, mudando a cor de uma pequena região retangular de uma imagem fornecida para processamento.

Alguns conceitos importantes serão abordados neste capítulo:

  • Tipos de dados comuns mais usados no OpenCV.

  • Funções para realizar entrada/saída de dados.

  • Funções para acessar os pixels de uma imagem.

  • Funções para realizar interações na interface gráfica.

Para experimentar a manipulação dos pixels em uma imagem, realize a compilação do programa-exemplo pixels.cpp, mostrado na Listagem 4, “pixels.cpp”, e teste-o com a imagem bolhas.png. Reproduza o esquema de preparação do arquivo executável com CMake, compile o programa-exemplo e execute-o. Ele deverá abrir a imagem bolhas.png (interpretando-a em escala de cinza) e exibi-la em uma janela, desenhando um quadrado preto em uma região pré-estabelecida.

Após isso, ele irá aguardar que o usuário pressione alguma tecla. Uma vez pressionada a tecla, o programa reabrirá o arquivo da imagem interpretando-a agora em formato com cores e passará a desenhar um quadrado vermelho na mesma região que foi pré-estabelecida.

Listagem 4. pixels.cpp
#include <iostream>
#include <opencv2/opencv.hpp>

int main(int, char**) {
  cv::Mat image;
  cv::Vec3b val;

  image = cv::imread("bolhas.png", cv::IMREAD_GRAYSCALE);
  if (!image.data) std::cout << "nao abriu bolhas.png" << std::endl;

  cv::namedWindow("janela", cv::WINDOW_AUTOSIZE);

  for (int i = 200; i < 210; i++) {
    for (int j = 10; j < 200; j++) {
      image.at<uchar>(i, j) = 0;
    }
  }

  cv::imshow("janela", image);
  cv::waitKey();

  image = cv::imread("bolhas.png", cv::IMREAD_COLOR);

  val[0] = 0;    // B
  val[1] = 0;    // G
  val[2] = 255;  // R

  for (int i = 200; i < 210; i++) {
    for (int j = 10; j < 200; j++) {
      image.at<cv::Vec3b>(i, j) = val;
    }
  }

  cv::imshow("janela", image);
  cv::waitKey();
  return 0;
}

Para compilar e executar o programa-exemplo pixels.cpp, salve-o em um diretório juntamente com o arquivo CMakeLists.txt adaptado para o exemplo (Listagem 5, “CMakeLists.txt”), a imagem bolhas.png, e execute a seqüência de comandos mostrada na Listagem 6, “Compilação do programa pixels”.

Listagem 5. CMakeLists.txt
cmake_minimum_required(VERSION 3.0.0)
project(pixels VERSION 0.1.0 LANGUAGES C CXX)

find_package(OpenCV REQUIRED)
include_directories(${OpenCV_INCLUDE_DIRS})

add_executable(pixels pixels.cpp)
target_link_libraries(pixels ${OpenCV_LIBS})
Listagem 6. Compilação do programa pixels
$ mkdir build
$ cd build
$ cmake ..
$ make
$ cp ../bolhas.png .
$ ./pixels

2.1. Descrição do programa pixels.cpp

#include <opencv2/opencv.hpp>

Em geral, a maior parte das funcionalidades da biblioteca é definida no arquivo de cabeçalho opencv.hpp. É necessário procurar na árvore do sistema de arquivos onde ele está localizado para que seu caminho seja incluído no código. Normalmente, a definição apresentada neste exemplo é suficiente para que o compilador encontre o arquivo.

Pela inclusão desse arquivo, são definidos tipos básicos da biblioteca, bem como os protótipos de várias funções usadas para realiza o tratamento de imagens. Ele agrega as definições de funções e classes que são usadas para entrada e saída de dados, criação de tipos comuns e manipulação de janelas e seus eventos, além de widgets para permitir interação com o usuário.

cv::Mat image;

A interface C++ do OpenCV provê um tipo básico de estrutura para armazenar imagems: a classe cv::Mat. Dependendo da forma como é criado, um objeto dessa classe é capaz de armazenar imagens (matrizes) de diversos tipos diferentes, tais como int, char, float e double, além de algumas combinações desses tipos.

Perceba que o tipo cv::Mat é definido no namespace cv. O OpenCV define diversos namespaces para organizar as funcionalidades da biblioteca, e cv é o principal deles. É nele que se encontram as definições de tipos e funções mais comuns da biblioteca. Embora seja possível colocar no preâmbulo do programa a diretiva using namespace cv, é recomendável que se use o nome completo do namespace para evitar conflitos com outros namespaces que possam ser usados.

Dezenas de métodos são providos para a classe cv::Mat, métodos estes que poderão ser apresentados ao longo do texto, conforme a demanda ocorra. Nesta lição, apenas o método at será utilizado, posto que é um dos que permite o acesso aos pixels da imagem de forma individual.

Outro tipo apresentado no exemplo - cv::Vec3b - encontra-se entre vários outros que são predefinidos no OpenCV. A classe cv::Vec que serve de base para a criação de cv::Vec3b é definida para abrigar diversas formas de vetores curtos, como aqueles necessários para armazenar as informações de cor de um pixel individual. A classe é definida por gabaritos e provê armazenamento de uma quantidade de valores de um dado tipo fornecido na instanciação do gabarito. Para ilustrar o uso da classe cv::Vec, alguns dos tipos predefinidos no OpenCV são mostrados a seguir.

typedef Vec<uchar, 2> Vec2b;
typedef Vec<uchar, 3> Vec3b;
typedef Vec<uchar, 4> Vec4b;

typedef Vec<short, 2> Vec2s;
typedef Vec<short, 3> Vec3s;
typedef Vec<short, 4> Vec4s;

typedef Vec<int, 2> Vec2i;
typedef Vec<int, 3> Vec3i;
typedef Vec<int, 4> Vec4i;

typedef Vec<float, 2> Vec2f;
typedef Vec<float, 3> Vec3f;
typedef Vec<float, 4> Vec4f;
typedef Vec<float, 6> Vec6f;

typedef Vec<double, 2> Vec2d;
typedef Vec<double, 3> Vec3d;
typedef Vec<double, 4> Vec4d;
typedef Vec<double, 6> Vec6d;

No caso do exemplo em discussão, o tipo cv::Vec3b usado representa um vetor de três componentes, cada uma do tipo unsigned char, ocupando apenas três bytes na memória. Esse tipo é normalmente usado para representar pixels em imagens coloridas, onde cada pixel é representado pelas suas componentes de vermelho, verde e azul (RGB).

image= cv::imread("bolhas.png",cv::IMREAD_GRAYSCALE);
if(!image.data)
  std::cout << "nao abriu bolhas.png" << std::endl;

Esse trecho de código usa a função cv::imread() para ler uma imagem presente em um arquivo e armazená-la no objeto image. A função cv::imread() recebe dois parâmetros: o primeiro deles é o caminho para o arquivo a ser aberto; o segundo é a forma como a imagem será interpretada. Neste caso, independentemente do formato da imagem presente no arquivo bolhas.png, ela será imediatamente transformada em uma imagem em tons de cinza antes de ser guardada no objeto image. Como o segundo argumento é cv::IMREAD_GRAYSCALE a leitura da imagem é forçada com posterior conversão para tons de cinza.

Caso o arquivo não seja aberto (ex: o arquivo não foi encontrado, ou o usuário não possui permissão de leitura ativado), o campo data do objeto image deverá ter valor nulo, sinalizando o erro.

Algo interessante a ser observado nesse momento é que algumas das estruturas internas da classe cv::Mat são públicas. Isso ajuda a não haver degradação de desempenho com o uso excessivo de chamadas de métodos, por exemplo, em leitura de fluxos de vídeo.

std::namedWindow("janela", cv::WINDOW_AUTOSIZE);

Cria uma janela para que o usuário possa referenciá-la pelo nome que é fornecido. O parâmetro cv::WINDOW_AUTOSIZE permitirá que a janela se ajuste automaticamente para o tamanho da imagem que for fornecida para exibição.

for(int i=200;i<210;i++){
  for(int j=10;j<200;j++){
    image.at<uchar>(i,j)=0;
  }
}

Este trecho de código desenha um retângulo preto na imagem. Neste caso, assumindo que a primeira dimensão representa a coordenada x e a segunda dimensão da matriz representa a coordenada y, será desenhado um retângulo do ponto \((200,10)\) até o ponto \((210,200)\).

Os pixels da região são acessados ou modificados com o método at. Observe que este método sofre o efeito de um gabarito, que deve receber um tipo correspondente o tipo de dado que está armazenado no objeto image. Como a leitura nesta parte do código foi feita assumindo uma imagem em tons de cinza, o tipo de dado presente na memória será neste caso unsigned char. Tipos diferentes poderão gerar resultados incorretos, posto que a função at() interpretará a sequência de bytes da matriz image de forma inapropriada.

Para manter os sistema referencial do tipo destrógiro, assume-se que os eixos x e y ficarão organizados conforme apresenta a Figura 3, “Sistema referencial adotado no OpenCV.”, com a origem no canto superior esquerdo da imagem.

Figura 3. Sistema referencial adotado no OpenCV.
cv::imshow("janela", image);
cv::waitKey();

A imagem image é mostrada na janela "janela" e o programa aguarda até que o usuário digite alguma tecla.

image= cv::imread("bolhas.png", cv::IMREAD_COLOR);

O procedimento agora repete os passos anteriores, só que dessa vez a imagem é interpretada com três componentes de cor em virtude do parâmetro cv::IMREAD_COLOR. A matriz image agora guardará, para cada pixel, um conjunto de 3 bytes para armazenar as contribuições de vermelho, verde e amarelo que este possui, inclusive se a imagem fornecida for em tons de cinza.

val[0] = 0;   //B
val[1] = 0;   //G
val[2] = 255; //R

As cores em um pixel são ordenadas na sequência B→G→R (Azul, Verde, Vermelho), sendo agora os pixels da matriz serem interpretados como um conjunto de elementos do tipo cv::Vec3b.

for(int i=200;i<210;i++){
  for(int j=10;j<200;j++){
    image.at<Vec3b>(i,j)=val;
  }
}

Agora, a função cv::at() interpreta a sequência de bytes usada para armazenar a matriz image como sendo formadas por pixels do tipo cv::Vec3b, sendo às posições correspondentes atribuídas a cor vermelha predefinida na variável val.
