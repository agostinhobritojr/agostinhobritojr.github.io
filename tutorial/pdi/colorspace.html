Capítulo 7. Modelos de Cor

O objetivo dessa lição é apresentar modelos de cores e como podem ser usados para destacar aspectos em uma imagem.

Espaços de cores são modelos matemáticos que descreve como a cor pode ser representada em um computador. Os modelos podem ser orientados a hardware ou orientados à percepção humana. O modelo mais comum usado na representação de imagens é o modelo RGB (Red, Green, Blue). Trata-se de um modelo de cor aditivo, baseado na composição da luz, onde as cores são formadas pela combinação de vermelho, verde e azul. O modelo RGB é usado em dispositivos de exibição, como monitores e televisores, e em câmeras digitais.

Já em dispositivos de impressão, utiliza-se o modelo CMYK (Cyan, Magenta, Yellow, Black), que é um modelo de cor subtrativo, baseado na absorção da luz, posto que as cores são formadas pela mistura de pigmentos que absorvem a luz.

O modelo de cores HSV (Hue, Saturation, Value) é um modelo de cor que descreve a cor de forma mais intuitiva para o ser humano, pois separa em suas componentes as noções de matiz (Hue) - o comprimento de onda dominante da cor - a saturação, que descreve a intensidade da cor (ou o quanto esta se encontra diluída no branco) e o valor, associada com a luminosidade, ou brilho da cor.

Os modelos possuem mecanismos de conversão entre si, de sorte que é possível representar imagens em diferentes modelos, conforme a necessidade.

O exemplo da Listagem 22 abre uma imagem colorida e decompõe seus canais em dois modelos distintos: RGB e HSV.

Listagem 22. colorspace.cpp
#include <iostream>
#include <opencv2/opencv.hpp>
#include <vector>

int main(int argc, char** argv){
  cv::Mat image;
  cv::Mat hsv, rgb, h, s, v;

  std::vector<cv::Mat> planes;
  int camera;
  double min, max;

  if(argc != 2){
    std::cerr << "Uso: " << argv[0] << " <Image_Path>\n";
    return -1;
  }

  image = cv::imread(argv[1], cv::IMREAD_COLOR);

  if (!image.data) {
    std::cerr << "Nao abriu " << argv[1] << std::endl;
    return -1;
  }

  cv::namedWindow("image", cv::WINDOW_NORMAL);
  cv::imshow("image", image);

  cv::split (image, planes);
  cv::hconcat(planes[0], planes[1], rgb);
  cv::hconcat(rgb, planes[2], rgb);
  cv::imshow("rgb", rgb);
  
  cv::cvtColor(image, image, cv::COLOR_BGR2HSV_FULL);
  cv::split (image, planes);
  cv::hconcat(planes[0], planes[1], hsv);
  cv::hconcat(hsv, planes[2], hsv);
  cv::imshow("hsv", hsv);

  cv::applyColorMap(planes[0], h, cv::COLORMAP_HSV_FULL);
  cv::imshow("h", h);

  cv::waitKey();
}

Para compilar e executar o programa-exemplo colorspace.cpp, salve-o em um diretório juntamente com o arquivo CMakeLists.txt adaptado para o exemplo (Listagem 23), a imagem frutas.png e execute a seqüência de comandos mostrada na Listagem 24.

Listagem 23. CMakeLists.txt
cmake_minimum_required(VERSION 3.0.0)
project(colorspace VERSION 0.1.0 LANGUAGES C CXX)

find_package(OpenCV REQUIRED)
include_directories(${OpenCV_INCLUDE_DIRS})

add_executable(colorspace colorspace.cpp)
target_link_libraries(colorspace ${OpenCV_LIBS})
Listagem 24. Compilação e execução do programa colorspace
$ mkdir build
$ cd build
$ cmake ..
$ make
$ ./colorspace ../frutas.png

7.1. Descrição do programa colorspace.cpp

cv::cvtColor(image, image, cv::COLOR_BGR2HSV_FULL);
cv::split (image, planes);

O programa colorspace.cpp realiza a leitura de uma imagem de entrada, seguido da sua decomposição em canais RGB e HSV. A decomposição nos canais RGB é feita diretamente usando a função cv::split(), enquanto a decomposição nos canais HSV é feita somente após a conversão da imagem para o modelo HSV. Neste último caso, a conversão é feita com o parâmetro cv::COLOR_BGR2HSV_FULL, que gera uma imagem de saída em 8 bits por canal, com valores de 0 a 255.

cv::applyColorMap(planes[0], h, cv::COLORMAP_HSV);

A função cv::applycolormap() aplica um mapa de cores a uma imagem de entrada. O primeiro argumento é a imagem de entrada, o segundo argumento é a imagem de saída e o terceiro argumento é o mapa de cores a ser aplicado. O OpenCV fornece vários mapas de cores, como cv::COLORMAP_JET, cv::COLORMAP_PARULA, cv::COLORMAP_SPRING, entre outros.

Os mapas devem ser escolhidos de acordo com a aplicação e o tipo de imagem que se deseja apresentar e, apesar de serem úteis para destacar detalhes em imagens em tons de cinza, é importante lembrar que a cor não deve ser usada como única informação para a interpretação de dados e que o dado mascarado pela cor ainda é a imagem em tons de cinza.
