Capítulo 18. Limiarização

A limiarização (ou Threshold) é uma técnica de processamento de imagens que visa separar os pixels de uma imagem em categorias distintas. As técnicas mais comuns lidam com imagens em tons de cinza, onde a separação é feita em duas categorias: pixels que possuem intensidade maior que um limiar e pixels que possuem intensidade menor que o limiar. Nesse caso, a imagem resultante de um processo de limiarização de uma imagem em tons é uma imagem binária, onde os pixels são representados por valores de intensidade 0 ou 255.

A limiarização é uma técnica simples e eficaz para segmentar imagens. Entretanto, a escolha do limiar é uma tarefa que pode ser difícil, principalmente quando a imagem possui variações de iluminação ou ruído. A escolha do limiar pode ser feita de forma automática, usando técnicas de otimização, ou de forma interativa, onde o usuário ajusta o limiar até obter o resultado desejado.

A biblioteca OpenCV possui uma função que implementa a limiarização de imagens, que serve tanto para o caso manual quanto para o automático. A função cv::threshold() recebe como argumentos a imagem a ser processada, a imagem onde o resultado será gravado, o valor do limiar, o valor máximo que um pixel pode assumir e o tipo de limiarização.

O programa de referência utilizado para essa tarefa, threshold.cpp, é mostrado na Listagem 54, “threshold.cpp”.

Listagem 54. threshold.cpp
#include <iostream>
#include <opencv2/opencv.hpp>

int main(int argc, char** argv) {
  cv::Mat image, image1;

  image = imread(argv[1], cv::IMREAD_GRAYSCALE);

  if (!image.data) {
    std::cout << "imagem nao carregou corretamente\n";
    return (-1);
  }

  cv::threshold(image, image1, 1, 255, cv::THRESH_BINARY + cv::THRESH_OTSU);

  cv::imshow("original", image);
  cv::imshow("otsu", image1);
  cv::imwrite("otsu.png", image1);

  cv::waitKey();
  return 0;
}

Para compilar e executar o programa-exemplo threshold.cpp, salve-o em um diretório juntamente com o arquivo CMakeLists.txt adaptado para o exemplo (Listagem 55, “CMakeLists.txt”), a imagem sementes1.png e execute a seqüência de comandos mostrada na Listagem 56, “Compilação e execução do programa threshold”.

Listagem 55. CMakeLists.txt
cmake_minimum_required(VERSION 3.0.0)
project(threshold VERSION 0.1.0 LANGUAGES C CXX)

find_package(OpenCV REQUIRED)
include_directories(${OpenCV_INCLUDE_DIRS})

add_executable(threshold threshold.cpp)
target_link_libraries(threshold ${OpenCV_LIBS})
Listagem 56. Compilação e execução do programa threshold
$ mkdir build
$ cd build
$ cmake ..
$ make
$ ./threshold ../sementes1.png

O programa irá gerar uma imagem chamada otsu.png com o resultado da limiarização da imagem sementes1.png usando o método de Otsu.

18.1. Descrição do programa threshold.cpp

Neste pequeno exemplo, apenas uma linha merece atenção especial:

cv::threshold(image, image1, 0, 255, cv::THRESH_BINARY | cv::THRESH_OTSU);

A função cv::threshold() recebe como argumento a imagem original na matriz image e grava o resultado na matriz image1, com valor de limiar igual inicial a 0 (usado apenas nos casos de threshold manual) e valor máximo de pixel igual a 255 (que será atribuído aos pixels que ultrapassam o valor do limiar atribuído ou calculado). O tipo de limiarização é definido pela operação cv::THRESH_BINARY + cv::THRESH_OTSU, que indica que o threshold será calculado automaticamente usando o método de Otsu, com saída binária.
