Capítulo 13. Registro de imagens

Considere a imagem da Figura 32, “Mostrador de um voltímetro digital com inclinação”. Trata-se de um mostrador de um voltímetro digital. Uma possível aplicação de visão computacional poderia ser capaz de capturar as imagens do mostrador e realizar o reconhecimento automático do valor de tensão mostrado no visor. Para isso, é necessário realizar a separação dos pixels que compõem o mostrador do voltímetro dos pixels que compõem o fundo da imagem.

Entretanto, a inclinação presente nesta imagem constitui uma distorção geométrica que pode dificultar a ação de uma ferramenta de reconhecimento de caracteres.

Figura 32. Mostrador de um voltímetro digital com inclinação

A correção da inclinação pode ser feita através de uma transformação de perspectiva. A transformação de perspectiva é uma técnica que permite a correção de distorções geométricas em imagens. A técnica consiste em mapear os pontos de uma imagem para uma nova posição, de forma que a imagem resultante tenha a perspectiva corrigida.

Um dos modelos possíveis para realizar a transformação é o modelo de transformação de perspectiva de quatro pontos. Neste modelo, são necessários quatro pontos de referência na imagem original e quatro pontos correspondentes na imagem corrigida. A transformação de perspectiva de um ponto é dada por uma matriz de transformação homogênea de dimensão 3x3 que pode ser calculada a partir dos pontos de referência e aplicada a todos os pontos da imagem.

\[\left[ \begin{array}{c} x' \\ y' \\ s \end{array} \right] = \left[ \begin{array}{ccc} h11 & h12 & h13 \\ h21 & h22 & h23 \\ h31 & h32 & 1 \\ \end{array} \right] \left[ \begin{array}{c} x \\ y \\ 1 \end{array} \right]\]

onde $(x, y)$ são as coordenadas do ponto na imagem original, $(x', y')$ são as coordenadas do ponto na imagem corrigida e $s$ é um fator de escala.

Com 4 pontos de referência é possível montar um sistema linear para obter os 8 coeficientes da matriz de transformação. No OpenCV, a função getPerspectiveTransform() realiza essa tarefa, permitindo obter a matriz de transformação a partir dos pontos de referência, e a função warpPerspective() aplica a transformação de perspectiva em toda uma imagem.

A Figura 33, “Mostrador de um voltímetro digital corrigido” mostra a imagem do voltímetro corrigida pela transformação de perspectiva usando essa técnica. Perceba que agora os dígitos não encontram-se mais inclinados.

Os quatro pontos que definem a região dos dígitos do mostrador foram determinados manualmente. São eles:

  • Ponto superior esquerdo: \((x=81, y=70)\);

  • Ponto superior direito: \((x=57, 324)\);

  • Ponto inferior esquerdo: \((x=210, y=311)\);

  • Ponto inferior direito: \((x=235, y=61)\).

Figura 33. Mostrador de um voltímetro digital corrigido

O programa de referência utilizado para essa tarefa, registro.cpp, é mostrado na Listagem 39, “registro.cpp”.

Listagem 39. registro.cpp
#include <opencv2/opencv.hpp>

int main(int argc, char *argv[]) {
  std::vector<cv::Point2f> srcPoints;
  std::vector<cv::Point2f> dstPoints;
  cv::Mat correctedImage;
  int width, height;

  cv::Mat image = cv::imread(argv[1]);
  if (image.empty()) {
    std::cout << "Imagem nao encontrada" << std::endl;
    return -1;
  }

  srcPoints.push_back(cv::Point2f(70, 81));    // superior esquerdo
  srcPoints.push_back(cv::Point2f(324, 57));   // superior direito
  srcPoints.push_back(cv::Point2f(311, 210));  // inferior direito
  srcPoints.push_back(cv::Point2f(61, 235));   // inferior esquerdo

  width = cv::max(cv::norm(srcPoints[0] - srcPoints[1]),
                  cv::norm(srcPoints[2] - srcPoints[3]));

  height = cv::max(cv::norm(srcPoints[1] - srcPoints[2]),
                   cv::norm(srcPoints[3] - srcPoints[0]));

  std::cout << "largura: " << width << std::endl;
  std::cout << "altura: " << height << std::endl;

  // Define os pontos de destino
  dstPoints.push_back(cv::Point2f(0, 0));
  dstPoints.push_back(cv::Point2f(width - 1, 0));
  dstPoints.push_back(cv::Point2f(width - 1, height - 1));
  dstPoints.push_back(cv::Point2f(0, height - 1));

  // Calcula a matriz de transformacao de perspectiva
  cv::Mat perspectiveMatrix = cv::getPerspectiveTransform(srcPoints, dstPoints);

  // Aplica a correção na imagem
  cv::warpPerspective(image, correctedImage, perspectiveMatrix,
                      cv::Size(width, height));

  // Exibe os resultados
  cv::imshow("Imagem original", image);
  cv::imshow("Imagem corrigida", correctedImage);
  cv::waitKey(0);

  return 0;
}

Para compilar e executar o programa-exemplo registro.cpp, salve-o em um diretório juntamente com o arquivo CMakeLists.txt adaptado para o exemplo (Listagem 40, “CMakeLists.txt”), a imagem voltimetro.png, e execute a seqüência de comandos mostrada na Listagem 41, “Compilação e execução do programa registro”.

Listagem 40. CMakeLists.txt
cmake_minimum_required(VERSION 3.0.0)
project(registro VERSION 0.1.0 LANGUAGES C CXX)

find_package(OpenCV REQUIRED)
include_directories(${OpenCV_INCLUDE_DIRS})

add_executable(registro registro.cpp)
target_link_libraries(registro ${OpenCV_LIBS})
Listagem 41. Compilação e execução do programa registro
$ mkdir build
$ cd build
$ cmake ..
$ make
$ ./registro ../voltimetro.png

A saída do programa registro é mostrada na Figura 33, “Mostrador de um voltímetro digital corrigido”.

13.1. Descrição do programa registro.cpp

srcPoints.push_back(cv::Point2f(70, 81));    // superior esquerdo
srcPoints.push_back(cv::Point2f(324, 57));   // superior direito
srcPoints.push_back(cv::Point2f(311, 210));  // inferior direito
srcPoints.push_back(cv::Point2f(61, 235));   // inferior esquerdo

A matriz de transformação necessita de dois conjuntos de pontos para ser calculada. O primeiro conjunto é composto pelos pontos de referência na imagem original e o segundo conjunto é composto pelos pontos correspondentes na imagem corrigida. Os pontos de referência na imagem original são armazenados no vetor srcPoints e os pontos correspondentes na imagem corrigida são armazenados no vetor dstPoints.

A sequência de pontos deve ser observada na ordem prevista para que a transformação seja realizada corretamente. A ordem dos pontos é Canto Superior Esquerdo → Canto Superior Direito → Canto Inferior Direito → Canto Inferior Esquerdo. Essa sequência deve ser observada tanto para os pontos de referência quanto para os pontos correspondentes.

width = cv::max(cv::norm(srcPoints[0] - srcPoints[1]),
               cv::norm(srcPoints[2] - srcPoints[3]));

height = cv::max(cv::norm(srcPoints[1] - srcPoints[2]),
               cv::norm(srcPoints[3] - srcPoints[0]));

dstPoints.push_back(cv::Point2f(0, 0));
dstPoints.push_back(cv::Point2f(width - 1, 0));
dstPoints.push_back(cv::Point2f(width - 1, height - 1));
dstPoints.push_back(cv::Point2f(0, height - 1));

A largura e altura da imagem de destino podem ser livremente definidas pelo usuário. No exemplo, a largura e a altura são estimadas a partir dos pontos de referência. A largura é dada pela maior distância entre os pontos superior esquerdo e superior direito e entre os pontos inferior esquerdo e inferior direito. A altura é dada pela maior distância entre os pontos superior direito e inferior direito e entre os pontos inferior esquerdo e superior esquerdo. Outras estratégias podem ser usadas para estimar essas medidas, mas dependem do conhecimento do usuário sobre o problema a ser abordado. Uma vez determinadas, os pontos correspondentes na imagem corrigida são armazenados no vetor dstPoints.

cv::Mat perspectiveMatrix = cv::getPerspectiveTransform(srcPoints, dstPoints);

cv::warpPerspective(image, correctedImage, perspectiveMatrix,
                    cv::Size(width, height));

A matriz de transformação de perspectiva é obtida e em seguida aplicada à imagem original. Perceba que o tamanho da imagem corrigida é definido pela largura e altura calculadas anteriormente. A imagem corrigida é armazenada na variável correctedImage e posteriormente exibida na tela.
