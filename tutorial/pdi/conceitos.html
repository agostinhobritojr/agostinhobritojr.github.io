Capítulo 1. Conceitos iniciais

1.1. O que é OpenCV

OpenCV (Open Source Computer Vision Library) é um projeto de código aberto disponível em http://www.opencv.org) que oferece uma biblioteca (ou conjunto de bibliotecas) com um vasto ferramental para tratamento de imagens, visão computacional e reconhecimento de padrões.

A biblioteca está disponível para algumas linguagens de programação e é organizada na forma de módulos, cada um agregando um conjunto de funções por categoria de uso. Entre os módulos disponíveis pela biblioteca, destacam-se:

  • Estruturas de núcleo, que agregam tipos de dados comuns, como matrizes e vetores, para armazenamento de informações de forma conveniente para os demais módulos.

  • Processamento de imagens, para realizar transformações geométricas, filtragem linear e não linear, tratamento de cor, entre outras coisas.

  • Vídeo, para avaliação de movimento (ex: análise de fluxo ótico) e rastreio de objetos.

  • Calibração de câmeras.

  • Extração de características.

  • Detecção de objetos.

  • Highgui, para tratamento facilitado na criação de interfaces gráficas.

Neste documento, alguns desses módulos serão explorados na forma de exemplos e exercícios de fixação.

Detalhes acerca da instalação da biblioteca OpenCV não são descritos aqui. Cada sistema operacional possui uma forma de instalação distinta, que pode ser consultatada no website da biblioteca.

1.2. Hello, OpenCV

O primeiro exemplo que será apresentado visa mostrar como compilar e executar um pequeno usando OpenCV. Em suma, será necessário o arquivo contendo o código-fonte a ser compilado e um arquivo contendo regras de compilação.

As tarefas de compilação foram automatizadas com a ferramenta CMake (www.cmake.org).

O CMake é uma ferramenta de código livre que ajuda no processo de construção, configuração e compilação de projetos de software. Com o CMake, o desenvolvedor consegue permitir que arquivos de compilação de lote denominados Makefiles sejam gerados automaticamente para diferentes plataformas, como Windows, Linux e macOS. Ainda, permite ajustar a inclusão de bibliotecas e dependências de forma simples e intuitiva, quando comparado com o uso de arquivos de compilação de lote.

O CMake é portável e independente de plataforma. Ele fornece uma maneira simplificada de descrever a estrutura do projeto e suas dependências, independentemente do sistema operacional ou do ambiente de desenvolvimento utilizado pelo desenvolvedor. A criação de um projeto com CMake inicia com a preparação de um arquivo CMakeLists.txt, que contém as instruções para a construção do projeto. O CMake gera um arquivo Makefile a partir do arquivo CMakeLists.txt, que é usado para compilar o projeto.

Uma vez gerado o arquivo Makefile, o controle é repassado ao utilitário make, que decide automaticamente que partes de um projeto necessitam ser recompiladas. Para isso, invoca os comandos necessários para recompilá-las a partir da leitura das regras definidas no Makefile. Assim, para efetuar a compilação do programa, basta executar o comando make, ao invés invocar repetitivamente chamadas de compilador.

Em um projeto típico que utiliza OpenCV, o arquivo CMakelists.txt pode ser escrito conforme mostrado na Listagem 1. No arquivo, podem ser percebidos os comandos para localizar a biblioteca OpenCV e incluir os arquivos de cabeçalho e bibliotecas necessários para a compilação do programa. Ainda, é possível definir variáveis para armazenar o nome do programa a ser compilado e os arquivos fonte que serão utilizados.

Listagem 1. CMakeLists.txt
cmake_minimum_required(VERSION 3.0.0)
project(hello VERSION 0.1.0 LANGUAGES C CXX)

find_package(OpenCV REQUIRED)
include_directories(${OpenCV_INCLUDE_DIRS})

add_executable(hello hello.cpp)
target_link_libraries(hello ${OpenCV_LIBS})

O seu ambiente de desenvolvimento poderá ser testado com programa-exemplo hello.cpp , mostrada na Listagem 2, cuja única funcionalidade apresentar na tela uma imagem fornecida via linha de comando.

Listagem 2. hello.cpp
#include <iostream>
#include <opencv2/opencv.hpp>

int main(int argc, char** argv){
  cv::Mat image;
  image = cv::imread(argv[1],cv::IMREAD_GRAYSCALE);
  cv::imshow("image", image);
  cv::waitKey();
  return 0;
}

Para compilar e executar o programa-exemplo hello.cpp, salve-o em um diretório juntamente com o arquivo CMakeLists.txt e execute a sequência de comandos mostrados na Listagem 3. O primeiro comando cria um diretório denominado build e o segundo comando entra no diretório criado. O terceiro comando invoca o utilitário cmake para gerar o arquivo Makefile e o quarto comando invoca o utilitário make para compilar o programa. O último comando executa o programa, passando como parâmetro o nome de uma imagem que será exibida na tela.

Listagem 3. Compilação e execução do programa hello
$ mkdir build
$ cd build
$ cmake ..
$ make
$ ./hello ../biel.png

Caso o programa funcione conforme o exemplo, provavelmente seu ambiente de testes estará operacional para o restante dos exemplos.

Neste documento, geralmente imagens armazenada no formato PNG serão usadas. Este formato suporta representação de imagens de diversas formas, como em tons de cinza, coloridas e preto-e-branco. Imagens em outros formatos tais como JPEG podem oferecer algumas limitações para os casos que serão abordados, como perda de informação durante a compressão. Ainda, por exemplo, arquivos JPEG armazenam apenas imagens em formato colorido e, durante a o processo de gravação das imagens, o algoritmo de compressão com perdas pode modificar o conteúdo da imagem original a ser gravada, impedindo que alguns exemplos não funcionem conforme esperado.
