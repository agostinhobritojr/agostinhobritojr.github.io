3.1. Gerenciamento de layouts.

Mencionados na Seção 2.2, os layouts possuem funções importantes em qualquer aplicação. Eles são os responsáveis por organizar a distribuição dos objetos na janela da aplicação. Definindo os layouts nos certificamos que os componentes terão suas localizações bem definidas e que ao aumentar ou diminuir da janela do programa a distribuição de espaço será igual para todos os objetos.

Com o Qt Designer podemos criar layouts automáticos de forma bastante simples. Resumidamente, basta o desenvolvedor selecionar os objetos que pertencerão ao layout e escolher pelo tipo de distribuição desejada.

Figura 3.1. Opções de layouts oferecidos.


Todos os tipos de layouts possuem parâmetros de configuração comuns entre si bem como parâmetros específicos, tais parâmetros podem ser modificados através da tabela de propriedades do objeto.

3.1.1. Horizontal e Vertical layout.

Horizontal e vertical layouts são as formas mais simples de distribuição de objetos. No modo Horizontal layout os objetos contidos no layout são alinhados horizontalmente, já no modo Vertical layout o alinhamento é feito verticalmente.

Horizontal e vertical layouts podem ser combinados de inúmeras formas, por exemplo, um Horizontal layout pode conter vários Horizontais ou Verticais layouts internos.

Figura 3.2. Organização do layout horizontal e vertical.


Podemos adicionar Horizontais ou Verticais layouts ao programa através da lista de objetos, nela estão contidos os objetos Vertical Layout e Horizontal Layout.

Também podemos selecionar um conjunto de objetos que desejamos organizar em um layout e pressionar CTRL+H para alinhá-los horizontalmente ou CTRL+L para alinhá-los verticalmente.

A lista a seguir explica a função de cada parâmetro do objeto.

layoutName

Define o nome do layout.

layoutLeftMargin

Define a margem esquerda do layout.

layoutTopMargin

Define a margem superior do layout.

layoutRightMargin

Define a margem direita do layout.

layoutBottomMargin

Define a margem inferior do layout.

layoutSpacing

Define o espaçamento entre os componentes do layout.

layoutStretch

Determina o fator de espalhamento de cada componente do layout.

Exemplo de uso: layoutStretch: 0,1,2.

Para o exemplo mencionado, o segundo elemento do layout irá ocupar um espaço com um fator de 1, ou seja o espaço que este componente deverá tomar do layout será maior que o primeiro e menor que o terceiro componente. Para o terceiro componente do layout, o seu espaço será muito maior que o primeiro e maior que o segundo elemento. Todos os fatores iguais farão com que a divisão de espaço seja igualitária.

layoutSizeConstraint

Este parâmetro define o tamanho dos componentes internos ao layout. Podemos então estabelecer se o principal componente interno do layout irá ter o tamanho maximizado, minimizado ou inerente ao tamanho do layout.

3.1.2. Horizontal e Vertical lay out com separador.

Este tipo de distribuição é similar ao anterior, porém neste caso o usuário tem a possibilidade de definir por qual componente da distribuição terá maior espaço, ou seja os separadores poderão ser clicados e arrastados de modo que o elemento desejado do layout irá obter maior espaço na distribuição.

Podemos selecionar layouts com separadores a partir de dois tipos: Horizontal ou Vertical. No modo horizontal, os elementos e os separadores são alinhas horizontalmente, já no modo vertical os elementos e os separadores são alinhados verticalmente.

Figura 3.3. Organização do layout horizontal e vertical com separadores.


A lista a seguir explica a função de cada parâmetro do objeto.

Nota

O layout com separador é um objeto herdado do QFrame, logo alguns parâmetros de configuração não são exclusivos do layout com separador. A lista que segue exibe somente os parâmetros exclusivos do objeto em questão.

orientation

Define a orientação do layout (horizontal ou vertical).

opaqueResize

Habilita ou não a exibição da prévia de alocação de espaço dos objetos ao ato de redimensionar os objetos do layout provenientes do usuário.

handleWidth

Define a largura dos separadores.

childrenCollapsible

Habilita ou não a possibilidade de redimensionalização dos objetos do layout para o valor de 0, isto é possibilitando a invisibilidade de alguns componentes do layout.

3.1.3. Layout em formato de formulário.

Este é um tipo especial de layout, é utilizado quando temos um conjunto de objetos que possuem uma função similar a um formulário ou questionário. Este formato faz com que sejam criadas duas colunas no layout, na primeira são posicionados os QLabels, na segunda são alocados os widgets(Line Edit, Spin Box e etc.) referentes a cada QLabel. Desta forma a organização se torna ideal visto que todas as questões de espaçamentos e posicionamentos são sanadas dado que a distribuição dos objetos segue um modelo padrão.

Figura 3.4. Organização do layout em formato de formulário.


Para adicionar um layout com formato de formulário utilize a lista de objetos pesquisando pelo componente Form Layout. Com o Form Layout criado, dê dois clique com o botão esquerdo do mouse sobre o novo componente. Na nova janela exibida, forneça as seguintes informações: texto a ser escrito no QLabel que será criado (número 1 na Figura 3.5), o nome do QLabel (número 2 na Figura 3.5), o tipo de componente que o QLabel irá fazer referência (número 3 na Figura 3.5), o nome deste objeto referenciado (número 4 na Figura 3.5), a opção por definir ou não a relação de buddy do QLabel com o objeto (número 5 na Figura 3.5) e a linha em que o par de objetos antes configurados serão adicionados (número 6 na Figura 3.5). Clique em OK e perceba que um par de objetos composto por um QLabel e outro do tipo solicitado foi criado e já distribuído no layout.

Figura 3.5. Criação de uma nova linha ao layout em formulário.


Outra opção para criação do layout em questão é selecionando um conjunto de objetos em que se deseja distribuí-los em forma de formulário e clicando no botão Layout in a Form layout. O Qt SDK irá gerar automaticamente o layout especificado. Com o layout criado é possível ainda assim adicionar elementos um a um clicando e arrastando o objeto a ser adicionado até o layout, quando definido a posição correta do objeto no layout basta então liberar o clique do botão do mouse.

A lista a seguir explica a função de cada parâmetro do objeto.

layoutName

Define o nome do layout.

layoutLeftMargin

Define a margem esquerda do layout.

layoutTopMargin

Define a margem superior do layout.

layoutRightMargin

Define a margem direita do layout.

layoutBottomMargin

Define a margem inferior do layout.

layoutHorizontalSpacing

Define o espaçamento horizontal entre os componentes do layout.

layoutVerticalSpacing

Define o espaçamento vertical entre os componentes do layout.

layoutFieldGrowthPolicy

Define o comportamento de crescimento das células do layout.

layoutRowWrapPolicy

Define o comportamento da linha do layout que contém um par de QLabel e widget ao ajuste de tamanho.

layoutLabelAlignment

Define o alinhamento dos objetos pertencentes a coluna da esquerda do layout (coluna dos QLabel).

layoutFormAlignment

Define o alinhamento dos objetos pertencentes a coluna da direita do layout (coluna dos widgets).

layoutSizeConstraint

Este parâmetro define o tamanho dos componentes internos ao layout. Podemos então estabelecer se o principal componente interno do layout irá ter o tamanho maximizado, minimizado ou inerente ao tamanho do layout.

3.1.4. Layout em forma de grade.

Um caso mais genérico de layouts em formato de formulário é o caso de layouts distribuídos em grade. Neste caso podemos criar um layout com inúmeras colunas e inúmeras linhas.

Figura 3.6. Organização do layout em forma de grade.


Este tipo de organização nos da mais liberdade para distribuir os componentes no layout ao nosso gosto, entretanto força o layout a ser menos flexível, isto é, sempre serão gerados layouts do mesmo formato, retangulares.

Para determinadas aplicações é mais aconselhável utilizar este tipo de organização ao invés de simplesmente combinar vários Horizontais e Verticais layouts entre si visto que em formato de grade a estrutura já está toda pré-definida.

Para criarmos um layout em forma de grade selecionamos os objetos que desejamos distribuir e então clicamos no botão Lay Out in a Grid ou pressionamos o comando CTRL+G. Outra forma de criar um layout em formato de grade é utilizando a lista de objetos, nela está contido o componente Grid Layout. Para adicionar um componente a um Grid Layout já criado basta clicar e arrastar o componente de interesse até a posição desejada na grade.

A lista a seguir explica a função de cada parâmetro do objeto.

layoutName

Define o nome do layout.

layoutLeftMargin

Define a margem esquerda do layout.

layoutTopMargin

Define a margem superior do layout.

layoutRightMargin

Define a margem direita do layout.

layoutBottomMargin

Define a margem inferior do layout.

layoutHorizontalSpacing

Define o espaçamento horizontal entre os componentes do layout.

layoutVerticalSpacing

Define o espaçamento vertical entre os componentes do layout.

layoutRowStretch

Determina o fator de espalhamento de cada linha do layout. A analogia é válida para o parâmetro layoutStretch do Vertical layout.

layoutColumnStretch

Determina o fator de espalhamento de cada coluna do layout. A analogia é válida para o parâmetro layoutStretch do Horizontal layout.

layoutRowMinimumHeight

Define a altura mínima para cada linha do layout. Por exemplo: layoutRowMinimumHeight: 0,1,3. No exemplo dado a primeira linha poderá ter a altura 0, a segunda poderá ter no mínimo a altura 1 e a terceira poderá ter no mínimo a altura 3.

layoutColumnMinimumWidth

Define a largura mínima para cada coluna do layout.

Por exemplo: layoutColumnMinimumWidth: 0,2,4.

No exemplo dado a primeira coluna poderá ter a largura 0, a segunda poderá ter no mínimo a largura 2 e a terceira poderá ter no mínimo a largura 4.

layoutSizeConstraint

Este parâmetro define o tamanho dos componentes internos ao layout. Podemos então estabelecer se o principal componente interno do layout irá ter o tamanho maximizado, minimizado ou inerente ao tamanho do layout.

3.1.5. Break layout e Adjust Size.

O botão Break Layout nos permite desmontar layouts já criados. Para isto, selecione um layout o qual deseja desmontar e clique no botão em questão.

O opção Adjust Size minimiza o objeto o máximo possível garantindo que todos os objetos interno tenham espaço suficiente para exibir seus conteúdos. Para executar esta opção, selecione um conjunto de objetos de interesse e clique no botão Adjust Size ou simplesmente pressione CTRL+J.

Nota

Observe que o botão Adjust Size é funcional para qualquer objeto/componente do programa. Sendo assim, os layouts por serem objetos/componentes do programa também podem ser ajustados pelo botão em questão.

3.2. Adição/Remoção de Signals e Slots de forma manual.

De acordo com o Capítulo 2 é possível criar Signals e Slots para uma classe do seu programa (MainWindow ou outras que foram criadas pelo desenvolvedor), com estes Signals e Slots possuímos mais poder de gerenciamento da aplicação. Porém, se o desenvolvedor desejar criar Signals e Slots via código e estabelecer conexões através do Editor de Signals/Slots não obterá resultados pois os Signals e Slots antes criados não estarão presentes na lista de Signals/Slots exibida.

Mediante tal problema a solução está em criar os respectivos Signals/Slots no Editor de Signals/Slots do Qt Designer. A seguir é listado o procedimento necessário para adicionar ou remover Signals/Slots manualmente.

  1. Criando Signals/Slots em código.

    Estando com o código do programa aberto, insira ao header (extensão .h) o Signal ou Slot que deseja criar. Para definir um método como um Signal basta digitar o que se segue.

    signals:
	  void meu_signal();

    Desta forma, o Signal meu_signal foi definido. Para definir-mos Slots temos duas opções: private ou public.

    private slots:
	  void meu_slot_private();
public slots:
	  void meu_slot_public();

    Se o Slot que deseja criar é conhecido por todas as classes então ele deve ser um public slots, caso somente as classes internas tenham conhecimento do Slot então este deve ser um private slots.

    Nota

    Note que é recomendável a implementação do Slot no .cpp, isto é, os Signals podem ser somente declarados e não implementados, já os Slots podem ser declarados e implementados no arquivo fonte (.cpp).

  2. Criando Signals/Slots via Qt Designer.

    Após ter criado o Signal/Slot no código fonte, devemos defini-lo também na lista de Signals/Slots do Qt Designer.

    Com o arquivo de interface gráfica (extensão .ui) aberto no Qt Designer, clique com o botão direito do mouse sobre a tela do MainWindow, selecione a opção Change signals/slots....

    Figura 3.7. Signals/Slots do MainWindow.


    Na nova janela exibida temos a lista de todos os Signals/Slots do MainWindow conhecidos pelo Qt Designer. Note que existem botões com os símbolos + e - tanto para Signals quanto para Slots. Através destes botões adicionamos e removemos os Signals/Slots manualmente.

    Fato importante a se observar é que o nome do Signal ou Slot criado em código deve ser exatamente igual ao criado através do Qt Designer, isto é, se o desenvolvedor criou um Signal com a seguinte estrutura.

    signals:
	  void signal_botao5(bool b,QString texto);

    A criação através do Qt Designer também deverá ser igual. Logo, o desenvolvedor irá clicar no botão + do campo de Signals e irá digitar o que se segue.

    void signal_botao5(bool,QString)

    Em seguida irá clicar em OK para confirmar a alteração.

Façamos um exemplo, um programa constituído de dois Slots e um Signal, os Slots serão executados ao clique dos botões, o Signal será o responsável por habilitar ou não um dos botões. Segue a interface do programa.

Figura 3.8. Interface do nosso exemplo.


Em resumo, o programa possuirá dois botões: Verificar e Submeter. O botão Verificar estará responsável por analisar os campos que devem ser preenchidos pelo usuário. O clique do botão estará conectado com um dos Slots manualmente criado. O botão Submeter estará inicialmente desabilitado, este é responsável por obter os dados fornecidos pelo o usuário e exibir tais informações em um espaço pré-definido (QLabel).

O objetivo do programa é forçar a criação de Signals/Slots manualmente e utilizá-los na arquitetura do programa.

Crie um novo projeto e abra o arquivo header do novo programa, neste arquivo iremos declarar todos os Signals/Slots da aplicação.

signals:
      void HabilitaSubmeter(bool);
public slots:
      void RecebeCliqueVerifica();
      void RecebeCliqueSubmeter();

Tendo declarado os Signals/Slots, abra o arquivo .ui através do Qt Designer e adicione à janela principal do projeto todos os componentes necessário para a aplicação (Figura 3.8). Adicionado todos os componentes, clique com o botão direito do mouse sobre a tela principal do programa e adicione os Signals/Slots criados via código através da opção Change signals/slots....

Figura 3.9. Signals/Slots adicionados.


Com isto, pareamos os Signals/Slots do código com os Signals/Slots do Qt Designer. Utilize o modo editor de Signals/Slots do Qt Designer e efetue as conexões de acordo com a Figura 3.10

Nota

A conexão do Signal HabilitaSubmeter(bool) com o Slot setEnabled(bool) só foi possível pois ao criar a conexão através do editor de Signals/Slots habilitamos o checkbox que permite exibir os Signals/Slots herdados de QWidget.

Figura 3.11. Habilitando a exibição de Signals/Slots herdados de QWidget.


As conexões feitas foram:

Figura 3.12. Tabela de conexões Signals/Slots estabelecidas.


Estabelecida as conexões de Signals/Slots devemos implementar os Slots utilizados, abra o arquivo .cpp da classe MainWindow e adicione o seguinte trecho de código.

void MainWindow::RecebeCliqueVerifica()
{
    bool nomeVazio = ui->nomeLineEdit->text().isEmpty();
    bool enderecoVazio = ui->enderecoLineEdit->text().isEmpty();
    int idade = ui->idadeSpinBox->value();

    if>(nomeVazio || enderecoVazio || idade<=0){
        emit HabilitaSubmeter(false);
    }else{
        emit HabilitaSubmeter(true);
    }
}

A função acima será executada a todo clique do botão Verificar. A função irá analisar se os campos de nome e endereço estão vazios (nomeVazio,enderecoVazio) e se o valor da idade fornecida é menor ou igual a 0 (idade). Se pelo menos uma das condições for verdadeira então a função irá emitir um Signal HabilitaSubmeter(false), isto é, o botão Submeter será desabilitado. Caso nenhuma das condições sejam verdadeiras a função irá emitir um Signal HabilitaSubmeter(true), agindo assim o botão Submeter será habilitado.

Tendo implementado o primeiro Slot referente ao clique do botão Verificar, podemos implementar o Slot referente ao clique do botão Submeter. Adicione ao arquivo fonte da classe MainWindow o seguinte trecho de código.

void MainWindow::RecebeCliqueSubmeter()
{
    QString nome = ui->nomeLineEdit->text();
    QString endereco = ui->enderecoLineEdit->text();
    int idade = ui->idadeSpinBox->value();

    QString texto = "NOME: "+nome+"\nENDEREÇO: "+endereco+"\nIDADE: "+QString::number(idade);
    ui->label->setText(texto);
    ui->label->adjustSize();
}

A função acima será executada a todo clique do botão Submeter. Ela irá obter o conteúdo nos campos em que o usuário fornece as informações e organizará para exibir tais informações no QLabel.

É necessário configurar o botão Submeter como desabilitado ao iniciar do programa, isto pode ser solucionado escrevendo o seguinte código no construtor da classe MainWindow.

ui->pushButton_submete->setEnabled(false);

Feito isto, construa e execute o programa. Perceba que as condições estabelecidas pelo botão Verificar são funcionais visto que o botão Submeter é habilitado somente no momento em que todos os campos estão preenchidos com informações.

3.3. Uso de arquivos de recursos.

Arquivos de recursos é uma ferramenta fornecida pelo Qt SDK que permite o desenvolvedor adicionar recursos (imagens, áudios, vídeos, códigos e etc) ao seu programa. Até o momento nossos programas foram feitos somente com textos adicionados via código, outros detalhes que podemos implementar ao programa é inserir ícones aos botões, background as janelas do programa e etc.

Nesta seção será demonstrado como criar arquivos de recursos e como utilizá-los no seu programa.

3.3.1. Criando arquivos de recursos.

Arquivos de recursos (.qrc) são arquivos similares aos códigos fontes (.cpp), headers (.h) e user interface (.ui), isto é, devem ser adicionados ao projeto criado.

Para adicionar um arquivo de recurso ao seu projeto clique com o botão direito sobre o nome do mesmo no navegador de projetos, em seguida selecione a opção Add New....

Figura 3.13. Opção Add New....


Após selecionado a opção, será exibida uma janela onde o desenvolvedor deverá selecionar o tipo do arquivo que será adicionado. Selecione a opção Qt e em seguida a opção Qt Resource file, clique em Choose... para confirmar a escolha.

Figura 3.14. Escolhendo tipo de arquivo a ser criado.


Uma nova janela será exibida, nela iremos fornecer informações sobre o nosso novo arquivo. Inicialmente, forneça ao programa o nome do novo arquivo e o local em que será criado (o local default fornecido é aconselhável). Clique em Next> para prosseguir.

Figura 3.15. Definindo nome do arquivo.


Na janela seguinte o programa requisitará ao desenvolvedor por qual projeto o arquivo criado será adicionado (o projeto default é aconselhável) e exibe a lista de arquivos que serão adicionados, neste caso somente o arquivo de recursos.

Figura 3.16. Confirmação de criação do arquivo.


Clique em Finish para completar a criação do arquivo.

Uma nova classe de arquivos Resources será adicionado ao seu projeto contendo o arquivo criado. Arquivos deste formato (.qrc) são constituídos basicamente por uma árvore de arquivos, isto é, o arquivo pode ser constituído de vários prefixos, cada prefixo pode ser preenchido com vários arquivos.

Figura 3.17. Estrutura do arquivo de recurso.


Antes de adicionar arquivos ao arquivo de recursos devemos criar pelo menos um prefixo ao mesmo. Sendo assim, com o arquivo de recursos aberto clique em Add e em seguida Add Prefix· Será listado ao arquivo um novo prefixo, altere o nome do mesmo no campo Prefix na parte inferior da janela. A partir de então é possível adicionar arquivos a este prefixo criado.

Para adicionar os arquivos selecione o prefixo desejado e então clique em Add e em seguida Add Files. Surgirá uma janela de navegação de diretório para que sejam selecionados os arquivos a serem adicionados ao prefixo.

Nota

Caso algum arquivo não esteja em um subdiretório do programa atual o Qt SDK irá sugerir que o desenvolvedor copie tal arquivo para um subdiretório do projeto.

Nota

O arquivo de recursos suporta alguns tipos de arquivos, normalmente são mais utilizados os arquivos de imagem visto que são bastante utilizados como ícones de botões por exemplo.

Figura 3.18. Exemplo de arquivo de recursos aberto com conteúdo.


Com os arquivos adicionados podemos declarar apelidos para cada arquivo, na parte inferior da janela, no campo alias definimos o apelido de cada arquivo de forma que facilite a sua identificação.

3.3.2. Utilizando arquivos de recursos.

No Qt SDK podemos inserir imagens ou arquivos externos utilizando o diretório de tal arquivo, porém este procedimento torna a programação muito restringida a casos exclusivos (computadores específicos). Com o arquivo de recurso podemos então compilar o programa junto com os arquivos, isto é, os arquivos inseridos nos recursos serão levados em conjunto com a própria aplicação.

Visto que o uso de arquivos de recursos é uma grande vantagem em termos de programação e organização temos que saber como utilizá-las em código. Em suma, utilizar um arquivo pertencente a um arquivo de recursos é similar ao uso de qualquer arquivo, ou seja, considerando que tenhamos um arquivo foto.png pertencente a um arquivo de recursos no prefixo imagens onde o apelido para o arquivo foto.png é photo podemos então referenciar tal arquivo das seguintes formas:

:imagens/foto.png

ou

:imagens/photo

Sendo assim, quando alguma função durante o desenvolvimento requisitar o diretório de algum arquivo externo basta adicionar tal arquivo ao arquivo de recursos e referenciá-lo da forma exemplificada acima.

No programa que segue foram desenvolvidos os conceitos de arquivos de recursos. Resumidamente foi criado um arquivo de recursos onde tal arquivo contém dois prefixos, cada prefixo contém duas imagens que são os ícones para dois botões. O programa tem por função alterar os ícones dos botões quando o usuário solicita a troca de tema do programa.

3.4. Criação e manipulação de ações.

QAction oferece ao programador a oportunidade de incluir ações ao seu programa, ações que podem ser geradas a partir de opções na barra de tarefas e em menus. É bastante comum em programas existir opções em menus e botões em barras de tarefas que executam a mesma função, eles estão vinculados a uma única ação e é isto que o QAction oferece.

Podemos adicionar ações ao nosso programa através de linhas de código ou utilizando o Qt Designer. A forma mais intuitiva e simples de se adicionar ações ao programa é utilizando a segunda opção pois neste caso basta o desenvolvedor digitar o nome da ação, inserir ícones se desejar e então adicionar conexões Signals/Slots ao mesmo.

3.4.1. Criando ações através do Qt Designer.

Crie um novo projeto e abra o arquivo mainwindow.ui, no Qt Designer localize o painel de propriedades dinâmicas do programa e então clique na aba Action Editor.

Figura 3.19. Painel de propriedades dinâmicas do programa.


Clique em New para criar um nova ação, a janela exibida solicitará informações básicas sobre a nova ação.

Figura 3.20. Criando uma nova ação.


No campo Text insira o texto referente a ação, no campo Object name será criado o nome da instância da classe QAction, no campo ToolTip o desenvolvedor tem a oportunidade de inserir dicas referente a ação, nos campos Icon theme e Icon o desenvolvedor poderá inserir ícones e temas a ação, na opção Checkable é requisitado se a ação é de caráter verificável ou apenas de botão e no campo Shortcut é inserido o atalho através do teclado para a ação. Ao fim da configuração clique em OK para confirmar a criação da ação.

Automaticamente uma ação será adiciona a lista do Action Editor contendo informações básicas sobre ela, para editá-la basta executar um duplo clique sobre a ação.

Tendo criado todas as ações é necessário criar no mínimo um QMenu ou QToolBar para adicionar as ações a eles. No Qt Designer dê um duplo clique onde está escrito Type Here na tela central do projeto e então digite o que desejar para o menu do programa. Note que com este passo criamos um menu para o programa onde podemos adicionar a ele várias ações. Para adicionar uma barra de tarefas ao programa clique com o botão direito sobre a tela central do projeto e então selecione a opção Add Tool Bar.

Figura 3.21. Criando menu e/ou adicionando novo barra de tarefas.


Com os locais para alocar as ações (menu ou barra de tarefas) devidamente criadas podemos então prosseguir com o projeto, clique e pressione em alguma ação criada no Action Editor e arraste até onde desejar na barra de tarefas ou menu, quando localizar a região libere o pressionamento. Desta forma criamos as ações e as alocamos no programa.

As ações possuem alguns Signals/Slots padrões referentes ao clique, mudança de estado e etc. Para fazer a conexão Signals/Slots com uma ação podemos utilizar a aba Signals & Slots Editor ou linhas de código com o comando connect.

Dica

Por se tratar de um botão na maioria dos casos de uso as ações normalmente são conectas com o Signal triggered().

O programa a seguir contém um exemplo de utilização de inúmeras formas, tanto de criação quanto de distribuição, de usar as ações de um programa.
