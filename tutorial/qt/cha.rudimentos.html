2.1. Conhecendo a interface do Qt SDK

O Qt SDK é uma poderosa ferramenta de programação que permite o desenvolvedor alterar o código livremente. Desta forma o programador tem capacidade de modificar todos os detalhes do programa criado. Por consequência de tal capacidade o Qt SDK possui uma interface grande e com bastantes opções onde muitas informações são fornecidas. Nesta seção serão explanadas as partes mais importantes da interface do Qt SDK.

Ao se criar um novo projeto, o Qt SDK lhe redireciona para o código fonte do seu projeto (arquivo de formato .cpp). À primeira vista são exibidas muitas informações ao redor do código. Veja a seguir a imagem de um projeto recém criado no Qt SDK.

Figura 2.1. Primeira tela após criação do projeto.


Na figura Figura 2.1 é exibido o modo de edição de código, é neste modo que são editados os códigos fontes do seu projeto, observe que a esquerda da janela existe um seletor de modos (região 1 da Figura 2.1), neste seletor é possível permutar entre os modos de utilização do Qt SDK. Cada modo possui uma função distinta:

Welcome (Boas Vindas)

Permite acessar projetos criados, abrir tutoriais, etc.

Edit (Edição)

Permite acessar os códigos fonte e edita-los.

Permite acessar e desenvolver aplicações com interface ao usuário.

É permitido somente com arquivos de interface gráfica (extensão .ui).

Permite que o desenvolvedor verifique os estados da sua aplicação enquanto depura o programa.

Permite configurar parâmetros de execução e compilação do projeto. Este modo é liberado somente quando o projeto está aberto.

Analyze (Análise)

Permite analisar o gerenciamento de memória e perfil do código enquanto executa o projeto.

Help (Ajuda)

Permite visualizar a documentação do Qt SDK.

Na barra lateral (região 3 da Figura 2.1) temos o navegador de projetos, esta barra é exibida somente nos modos de edição e depuração. No navegador o programador pode navegar entre os projetos e seus arquivos exibidos. É possível ainda criar conteúdos da barra lateral clicando em um dos botões Split (áreas em vermelho na região 3 da Figura 2.1) da barra, ao ser clicado uma nova subdivisão é gerada sendo esta passível de alteração de conteúdo clicando sobre o título do mesmo. As opções de conteúdo para as subdivisões da barra lateral são:

Para obter as classes básicas de uma classe, selecione a classe no código e clique com o botão direito do mouse sobre a seleção, por fim clique na opção Open Type Hierarchy (abrir tipos de hierarquia).

No modo designer a interface muda com relação ao modo de edição de código. Neste caso a programação ocorrerá graficamente. Na Figura 2.2 acima vemos que o modo de designer é constituído basicamente por seis regiões de funções distintas.

A região 1 da Figura 2.2 é a lista de objetos, a partir dela podemos inserir componentes ao nosso programa, tais componentes serão explicados na Seção 2.3.

Na região 2 da Figura 2.2 temos a tela central do projeto, é o espaço onde é exibido o layout do programa. Os objetos obtidos da lista deverão ser implementados na tela central.

Na região 3 da Figura 2.2 temos o painel de propriedades dinâmicas do programa, tais propriedades são Signals e Slots ou atalhos de teclado. Através deste painel podemos criar as conexões de Signal e Slots (melhores explanadas na Seção 2.2) e adicionar atalhos por comando de teclado ao programa.

A região 4 da Figura 2.2 contém a tabela de propriedade do objeto, esta fornece informações como nome, tipo, largura, altura de um objeto selecionado na tela central, também podemos definir valores default ao objeto selecionado.

Na região 5 da Figura 2.2 temos a árvore estrutural da janela central, esta exibe em estrutura de árvore ramificada a hierarquia dos objetos. Desta forma é facilitada a visualização de qual objeto está acima ou abaixo de outro.

A região 6 da Figura 2.2 é uma barra de ferramentas que permite alterar entre os quatro tipos de edição do modo designer e definir layouts para conjuntos de objetos da janela principal. Os quatro tipos de edição do modo designer são:

Edit Widgets

Neste modo é possível editar a janela principal do programa adicionando objetos, criando layouts e definindo propriedades a objetos.

Neste modo é possível criar conexões de Signal/Slot entre os objetos.

Como o nome diz, este modo permite alterar a ordem de foco dos objetos a partir do teclado (teclas SHIFT+TAB ou TAB pressionado).

Além dos tipos de edição a barra de ferramenta também permite criar layouts. Para criar um layout selecione um ou mais objetos da tela principal e clique em um dos modos permitidos, perceba que cada modo distribui os objetos de formas distintas. Depois de criado o layout é possível desmontá-lo clicando no botão Break Layout.

Com a familiarização estabelecida com o modo de edição e designer no Qt SDK estamos aptos desenvolver com mais facilidades aplicações com tal ferramenta.

2.2. Conceituando Signal e Slots

O Qt oferece aos seus desenvolvedores uma ferramenta muito útil, com ela podemos construir a arquitetura do nosso programa, ou seja definir como o mesmo se comporta mediante a ação do usuário do software. Tal ferramenta é o gerenciamento de Signals e Slots. No último exemplo do capítulo anterior quando implementamos uma ação ao clique do Pushbutton tínhamos construído uma conexão de Signal e Slot.

Signals e Slots permite de forma simples e intuitiva montar a estrutura do seu programa. Este mecanismo de comunicação intra objetos é o grande diferencial do Qt em relação aos outros frameworks.

Com Signals e Slots podemos informar a um objeto A que outro objeto B alterou seu estado, por exemplo no caso do nosso primeiro programa onde ao clicarmos no PushButton o texto do QLabel era alterado, neste caso tínhamos um Signal que era emitido automaticamente ao clicar no PushButton de forma que tal Signal estava conectado com um Slot da QMainWindow onde este tinha por função alterar o texto do QLabel.

Figura 2.3. Diagrama de funcionamento do Signals e Slots


Alguns Signals são padrões para alguns objetos, no PushButton por exemplo utilizamos o Signal padrão clicked() para ser emitido quando o botão for clicado, porém podemos criar Signals e emiti-los quando desejarmos com o comando emit. Similarmente, Slots também possuem suas instâncias padrões para certos objetos, que no nosso caso foi a função setText(QString) do objeto QLabel.

Nota

É uma prática comum o programador definir os Slots visto que somente ele saberá o que deverá ocorrer quando um certo Signal for emitido.

As conexões Signals e Slots podem ser feitas de diversos modos. Podemos emitir um Signal do Objeto A para vários Slots de outros objetos, emitir vários Signals de vários objetos para um único Slot do objeto A e emitir um Signal de um objeto A para um Signal de um objeto B (neste caso o Signal de A irá emitir o Signal de B sempre que o Signal de A for emitido).

Agora que sabemos a definição de Signals e Slots e como eles funcionam podemos criar nossas próprias conexões e assim montar a arquitetura do programa ao própio gosto. É possível criar conexões entre dois objetos de duas formas: por comandos e editor gráfico.

Por comando, se desejamos criar uma conexão entre o objeto A com a Signal sinalA() e o objeto B com o Slot slotB() devemos executar a seguinte linha de código.

connect(Objeto A,SIGNAL(sinalA()),Objeto B,SLOT(slotB()));

Desta forma quando o programa executar este comando uma nova conexão será criada entre os objetos e a partir de então quando o sinalA() do objeto A for emitido, automaticamente o slotB() do objeto B será executado.

Nota

Os Slots são funções normais, ou seja não necessita ser executada só e somente só com um Signal emitido, podemos fazer chamadas dele no código como uma função qualquer.

Com o Qt Designer (ferramenta que é agregado ao Qt Creator SDK) podemos criar conexões de forma gráfica, ou seja puxando e arrastando as conexões no sentido determinado. Para utilizar esta ferramenta crie um novo projeto e adicione a janela principal do programa os componentes Label e Push Button, clique em Editar Signals/Slots ou pressione F4.

Figura 2.4. Botão do editor de Signals e Slots.


Feito isto podemos então criar conexões diretamente na janela do programa, clique e arraste o mouse do PushButton até o QLabel na janela principal. Será exibida uma janela solicitando o Signal do PushButton e o Slot do QLabel, selecione o Signal clicked() e o Slot clear() e clique em OK. Desta forma o programa irá apagar o texto do QLabel quando o PushButton for clicado pois o Signal clicked() será emitido e consequentemente o Slot clear() será executado.

Figura 2.5. Conexão entre Signal e Slot estabelecida.


Figura 2.6. Definindo Signal e Slot da conexão.


Execute o programa e verifique que quando o PushButton é clicado o QLabel ficará sem texto. Esta foi uma das conexões possíveis criadas por padrão, podemos ainda criar nossos próprios Signals e Slots e conectá-las normalmente. Para criar uma conexão Signal e Slot não padrão do Qt via editor gráfico devemos criar os mesmos e adicioná-los ao editor.

Agora criaremos nosso Signal e iremos emitir quando quisermos, para isso crie um novo projeto. Com o projeto criado abra o arquivo mainwindow.ui e adicione a janela do programa um QLabel e um Horizontal Slider (para encontrá-los basta pesquisar no campo Filter).

Figura 2.7. Selecionando a opção Go To Slot... do Horizontal Slider.


Com os objetos posicionados, clique com o botão direito sobre o Horizontal Slider e selecione a opção Go To Slot..., isto irá disponibilizar as opções possíveis de Signals padrões do Horizontal Slider que será conectado com um Slot do QMainWindow, tal Slot será construído automaticamente pelo Qt SDK. Com a nova janela exibida clique na opção valueChanged(int), este Signal é emitido quando o valor do Horizontal Slider é alterado e o argumento passado é um inteiro respectivo ao novo valor. Clique em OK e o Qt irá criar o Slot automaticamente.

Agora que já temos a conexão para alteração de valor no Horizontal Slider vamos criar o Signal do QMainWindow, este será ativado quando o Horizontal Slider chegar a um certo valor. Abra o arquivo mainwindow.h e no escopo da classe MainWindow adicione a seguinte definição.

signals:
      void valor_atingido(QString aviso);

Isto criará o Signal valor_atingido(QString aviso) do QMainWindow, perceba que não é necessário criar implementação desta função Signal no arquivo .cpp porém é possível.

Com o Signal criado, volte ao arquivo Mainwindow.cpp na função Slot criada automaticamente pelo Qt SDK implemente o seguinte código.

void MainWindow::on_horizontalSlider_valueChanged(int value)
{
    if(value>=70){
        emit valor_atingido("Atenção!");
    }
}

Este código faz com que sempre que o valor do Horizontal Slider for alterado e o atual valor for maior que 70 o Signal valor_atingido("Atenção!") será emitido. Executando o programa a partir deste ponto nada acontecerá mediante a alteração do valor do Horizontal Slider pois o Signal é emitido entretanto nenhum Slot está conectado a ele.

Vamos criar agora uma conexão do Signal valor_atingido(QString) com o Slot setText(QString) do QLabel. Dê dois cliques no arquivo mainwindow.ui e abra o editor de Signals e Slots (pressionar F4). Visto que a conexão será entre o Signal do QMainWindow e o Slot do QLabel clique em qualquer lugar da janela principal e arraste até o QLabel.

Figura 2.8. Estabelecendo conexão entre o Signal do QMainWindow e o Slot do QLabel.


Feito isso uma janela solicitando o Signal do QMainWindow e Slot do QLabel será exibida, percebe-se que o Signal valor_atingido(QString) não é exibido, para isso clique no botão Edit... respectivo ao QMainWindow, surgirá uma nova janela listando os Signals e Slots do QMainWindow, na parte inferior (Signals) clique no botão com um símbolo de '+', apague o que foi escrito e digite:

valor_atingido(QString)

Pressione ENTER e clique em OK.

Figura 2.9. Configurando Conexão Signal e Slot.


Figura 2.10. Adicionando Signal ao QMainWindow.


Agora o Signal valor_atingido(QString) é exibido, clique nesta opção e em seguida será disponibilizado o Slot setText(QString), clique nesta opção também e então clique em OK.

Nota

Perceba que os Signals e Slots em sua maioria devem conter os mesmos parâmetros e na mesma ordem, exceto os Slots sem parâmetros, estes podem ser executados por qualquer Signal pois não necessitam de valores de entrada. Desta forma os valores que são emitidos junto ao Signal são repassados ao Slot.

Na janela principal será construída a conexão, sendo assim quando o Signal valor_atingido(QString) for emitido a função Slot setText(QString) do QLabel será executada.

Este trabalho poderia ser evitado utilizando o comando connect no construtor da classe MainWindow, este comando como explicado anteriormente faz a conexão entre os objetos, veja o comando para conectar o Signal valor_atingido(QString) do QMainWindow e o Slot setText(QString) do QLabel.

connect(this,SIGNAL(valor_atingido(QString)),ui->label,SLOT(setText(QString)));

O comando conecta o atual objeto this, no caso MainWindow, via Signal valor_atingido(QString) com o objeto ui->label, o QLabel, atráves do Slot setText(QString).

Para ambos os casos, criação de conexão via interface gráfica ou por comando, execute o programa, perceba que quando o Horizontal Slider atinge um valor maior ou igual a 70 o QLabel modifica o texto de TextLabel para Atenção!.

Figura 2.11. Projeto com Signal criado sendo executado.


Esta foi uma leve e consistente introdução sobre Signal e Slots, existem mais detalhes não menos importantes para esta ferramenta fundamental a qualquer programa desenvolvido no Qt SDK. Podemos perceber que para qualquer programa é necessário utilizar Signals e Slots, eles são a base do programa, a forma como o software se comportará mediante o usuário.

2.3. Componentes mais comuns

O Qt SDK possui muitos componentes gráficos bastante úteis para softwares em geral, tais componentes são: Botões, Caixa de texto, Scroll, CheckBox e etc. Todos possuem suas funções Slots e Signal com diversas funcionalidades. Nesta seção serão demonstrados os componentes mais utilizados em programas, iremos explanar sobre o que cada um faz, quais são os Slots e Signals mais comuns e explicar suas funcionalidades.

Nota

Para mais métodos destes componentes abaixo listados ou de outros componentes menos utilizados podemos utilizar a documentação oferecida pelo Qt SDK (modo de utilização Help (Ajuda)).

Botões são os componentes mais utilizados em qualquer programa, com eles podemos programar uma determinada ação a partir do clique do botão. Normalmente um botão esta associado a uma função Slot sensível ao clique porém no Qt SDK é possível configurar Slots para simples pressionamentos ou liberação do botão.
