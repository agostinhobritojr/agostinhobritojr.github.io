1.1. O que é o Qt (software)?

Qt é uma biblioteca de programação em C++ cujo principal objetivo é possibilitar o desenvolvimento de aplicações multiplataforma a partir de um mesmo código fonte. Ela foi concebida em 1991 por Haavard Nord e Eirik Chambe-Eng na Trolltech, empresa que era sediada na Noruega.

As primeiras versões do Qt permitiam o desenvolvimento em sistemas Windows© e Linux, e, na época, foi bastante popularizada pelo sistema de gerenciamentos de janelas Kde (www.kde.org). A evolução do Qt chegou a dispositivos móveis com Linux embarcado e symbian, chamando a atenção de empresas como a Nokia sobre as possibilidades da ferramenta. Em 2008, a Nokia obteve direitos de negócio sobre o Qt, havendo então se expandido para suportar diversos outros dispositivos criados por esta empresa. Em 2011, a Nokia desistiu do Qt, repassando a licença comercial da ferramenta para a Digia (www.digia.com). Em 2012, a Digia complementou os direitos obtendo todo o mercado do Qt Creator, sendo até hoje a detentora da marca.

Junto com a biblioteca Qt, seu SDK comporta diversos aplicativos que auxiliam no desenvolvimento de projetos. Entre estes, destaca-se o Qt Creator, um ambiente de desenvolvimento integrado que possibilita programação em Qt de forma eficiente e agradável. Com o Qt Creator é possível criar programas usando a linguagem de programação C++ em sintonia com elementos de programação visual. Em outras palavras, os programas podem ser desenvolvidos arrastando e soltando componentes gráficos como botões e caixas de texto usando o mouse, aumentando o rendimento e facilitando a tarefa de novos usuários se habituarem com o ambiente de desenvolvimento.

O Qt Creator vem suportando cada vez mais sistemas operacionais diferentes. Ao tempo de escrita desse documento encontrava-se disponíveis suportes para Linux, Windows, OS X, Android, iOS e QNX. Grandes empresas usam Qt para desenvolvimento, destacando-se exemplos famosos como VLC™ (player de mídia),Skype™,Google Earth™.

1.2. Preparando o ambiente de trabalho

O Qt possui diversas versões disponíveis, cada uma adequada a um tipo de ambiente e filosofia de desenvolvimento. Neste tutorial, serão descritos os principais passos para download e instalação do Qt nos sistemas operacionais Windows e Linux.

É importante notar que a biblioteca também é disponibilizada para desenvolvimento de aplicações comerciais, de sorte que o usuário deve verificar no website da Digia mais detalhes sobre o uso de licenças comerciais. Neste tutorial, o foco será dado à versão de código aberto do Qt.

O Qt SDK é um Software Development Kit (SDK), ou seja, um conjunto de aplicações que permitem a criação de programas de computador usando Qt. No SDK são incluídos a biblioteca Qt, o ambiente de desenvolvimento integrado Qt Creator, além de outros aplicativos que operam em conjunto para permitir a geração e compilação de código fonte em C++ com esta biblioteca.

Atenção

O Qt por si só não é um compilador, tratando-se apenas de uma biblioteca de programação. O compilador deve coexistir na máquina para que se possam produzir códigos executáveis usando Qt.

1.2.1. Preparação do Qt SDK no Windows

Acessando o site do Qt-Project (http://qt-project.org/), é possível chegar à área de downloads e escolher o instalador mais adequado para cada caso. É importante lembrar que as versões de instaladores online são menores, porém necessitam de conexão com a Internet durante a instalação. As versões completas são maiores, porém não necessitam de conexão com a internet.

Os exemplos expostos neste texto serão apresentados com base na instalação do Qt SDK no Windows® com a versão completa, posto que provê toda a parafernalha necessária para o programador. A instalação em versão online é bastante similar, de modo que os passos ilustrados a seguir serão úteis para ambas as instalações.

Podem ser encontradas diversas versões do instalador do Qt para Windows®. As versões geralmente contemplam o compilador utilizado e a arquitetura do sistema operacional. Duas categorias de versões são as mais comuns: aquelas que usam o compilador Visual Studio e as que usam o compilador MinGW. Caso o usuário disponha do Visual Studio, este deverá escolher a que melhor se adapta ao seus sistema. Caso não possua o compilador, melhor escolher a versão que vem com o MinGW incluído. Caso o usuário disponha dos compiladores nativos da Microsoft™ deve preferir o uso das versões para o Visual Studio, uma vez que o suporte para diversos recursos do sistema (ex: bibliotecas de reprodução de arquivos multimídia) somente está disponível para tais compiladores.

Após ter carregado e executado devidamente o instalador é iniciado o processo de instalação, a seguinte tela surgirá:

Figura 1.1. Tela de boas-vindas do instalador


Clique em Next para prosseguir a instalação. Em seguida é requisitado o local de instalação do Qt SDK, por padrão o instalador irá colocá-lo no disco C: porém isto pode ser alterado clicando em Browse.... O checkbox na parte inferior da janela seleciona se o Qt Creator deverá reconhecer arquivos do tipo .c,.h,.cc,.hh,.cxx,.hxx,.c++,.h++, .cpp e .hpp como arquivo padrão de execução.

Figura 1.2. Definindo diretório do Qt Creator


Clique em Next para prosseguir a instalação. Na janela seguinte será solicitado os componentes que se deseja instalar, por padrão o instalador irá instalar somente o Qt SDK e a ferramenta de compilação, no caso em exemplo será o MinGW 4.8, para se usar o Visual Studio como compilador deve-se carregar o instalador com VS na página de downloads do Qt-Project. Essas definições padrões são as básicas para a execução do Qt Creator porém aconselha-se a selecionar todos os componentes para evitar problemas futuros. Sendo assim, clique em Default para selecionar o componentes padrões, Select All para selecionar todos os componentes e Deselect All para selecionar nenhuma componente. É possível a todo momento visualizar no lado direito da janela o quanto será instalado no computador de acordo com a seleção dos componentes.

Figura 1.3. Seleção dos componentes para instalação


Clique em Next para prosseguir a instalação. Prosseguindo, o instalador irá se certificar se o usuário está de acordo com as licenças LGPL, FDL, GPL e exceções de LGPL que são em suma permissões, direitos e termos legais de uso do Qt SDK por parte do usuário. Caso o usuário esteja de acordo com as licenças deverá selecionar a primeira opção, caso contrário a única possibilidade será de finalizar a instalação.

Figura 1.4. Confirmação de conhecimento das licenças


Clique em Next para prosseguir a instalação. Feito isso, o instalador irá requisitar o apelido que será dado ao Qt SDK, por padrão o apelido será Qt <versão> sendo passível de alteração pelo usuário. Clique em Next para prosseguir a instalação. Após ter configurado todos os dados, o instalador irá fornecer um resumo do que e como será instalado de forma que o usuário esteja ciente das informações fornecidas, clique em Show Details para ver detalhes da instalação. Clique em Install para iniciar a instalação.

Figura 1.5. Resumo do que será instalado


Figura 1.6. Instalação sendo executada


Durante a instalação é possível ver o que está sendo feito clicando em Show Details, aguarde a instalação ser finalizada, isso poderá levar alguns minutos. Ao fim da instalação clique em Next, o instalador chegará a sua última parte onde o usuário deverá selecionar se deseja executar o Qt Creator ao finalizar a instalação e se deseja abrir o arquivo ReadMe do Qt Creator ao fim da instalação, este arquivo contém informações básicas sobre a versão do Qt Creator e sites para mais informações. O usuário deverá selecionar o que deseja e clicar em Finish. Desta forma o Qt SDK estará instalado no seu computador.

1.2.2. Qt SDK no Linux

É possível instalar o Qt SDK no Linux digitando apenas uma linha de código. Abra o terminal e digite o seguinte comando:

sudo apt-get install ubuntu-sdk

O sistema irá executar o comando e confirmar se o usuário deseja instalar o pacote. Confirme a instalação e aguarde o processo.

Feita a instalação basta executar o comando qtcreator no terminal e então o SDK será executado.

1.3. Primeiro Programa

Com o Qt SDK instalado podemos executar nosso primeiro programa. Seja em qual for o SO (Windows ou Linux) a forma de programação é a mesma. É aconselhável a se criar uma pasta para armazenar os programas criados a partir deste tutorial com o objetivo de facilitar o desenvolvimento do leitor e evitar possíveis erros de compilação.

Abra o Qt Creator, a tela de boas-vindas será exibida. Clique em Arquivo e selecione a opção Novo Arquivo ou Projeto, em seguida surgirá uma janela onde será possível seleciona qual tipo de arquivo ou projeto será iniciado.

Figura 1.7. Escolha por qual tipo de projeto ou arquivo


Selecione por Application, em seguida Qt Gui Application e então clique em Choose.... O Qt Gui Application permite que sejam criados programas com interface gráfica. Após clicado no botão Choose... será solicitado o nome e o local do projeto, digite o nome e escolha o local de criação do programa. O checkbox na parte inferior da janela quando habilitado faz com que o local de criação dos restantes dos programas criados pelo Qt Creator sejam pré-definidos no diretório definido anteriormente. Feito o preenchimento do nome e diretório do programa clique em Next> para prosseguir na instalação.

Figura 1.8. Definindo nome e diretório do projeto


Na janela seguinte o Qt Creator solicitará o local de criação das versões depuração e de lançamento do programa. Isto possibilita separar o código fonte do programa dos códigos de compilação gerados automaticamente e o executável que serão salvos em outras pastas. Por padrão habilite o programa para criar locais de depuração e lançamento e clique em Next>.

Figura 1.9. Criando diretórios de depuração e lançamento


Em sequência o Qt Creator solicitará informações sobre a classe que será criada, o tipo de classe que se deseja criar (QMainWindow, QWidget ou QDialog) o nome do arquivo header (.h), fonte (.cpp) e em caso de habilitado o nome do arquivo de form (.ui). Para início desejamos criar uma janela simples então deixamos tudo como padrão e no futuro estas opções serão melhor explanadas. Clique em Next> para prosseguir na criação do programa.

Figura 1.10. Definindo o nome da classe e dos arquivos


Por fim será exibida uma última janela de resumo, confirmando todos os passos anteriormente feito. Percebe-se que o Qt Creator confirma o diretório no qual será criado o programa e os arquivos nele criado. Caso esteja de acordo com os dados obtidos clique em Finish. O Qt Creator então iniciará a criação dos arquivos e irá exibi-los em uma árvore de diretórios.

Figura 1.11. Informações finais da criação do projeto


Figura 1.12. Projeto criado e apto ao desenvolvimento


Estando com o programa criado, é possível acessar a todos os arquivos sejam eles: headers (.h), fontes (.cpp),forms (.ui) e arquivo de extensão pro. Dê dois cliques no arquivo de form (.ui), será exibida a janela do programa em construção. Clique em Filter e digite Label, no visor logo abaixo estará o widget Label, clique e arraste até a janela central.

Figura 1.13. Posicionando o widget Label na janela


De forma similar digite em Filter por Push Button e arraste-o para a janela. A partir deste ponto o programa pode ser executado clicando em Run, o Qt Creator solicitará por salvar as alterações e então construirá o programa. Durante a execução nada ocorrerá pois não há nenhuma lógica ou evento implementado.

Figura 1.14. Posicionando o widget Pushbutton na janela


Figura 1.15. Primeiro programa sendo executado


Feche o programa e clique com o botão direito sobre o Push Button que está na janela principal, selecione Go To Slot..., na nova janela clique na opção Clicked() e então em OK. O Qt Creator irá implementar automaticamente uma função na classe do projeto referente ao evento de clique do botão. Dentro da função copie o seguinte código e clique em Run novamente.

 ui->label->setText("Hello!");

Desta vez o programa possui uma ação para o evento do clique no botão.

Figura 1.16. Primeiro programa sensível ao evento sendo executado


Este foi um programa simples, porém o Qt Creator tem mais funções para cada widget e muitas bibliotecas com inúmeras funcionalidades, em geral o Qt Creator possibilita a criação de programas bons e eficientes de forma muito prática.

1.4. Nosso projeto

Agora que já sabemos como instalar o Qt Creator e como construir um programa podemos definir uma ideia final de projeto.

Este tutorial terá por objetivo ensinar e demonstrar a implementação de ferramentas do Qt Creator com base na construção de um projeto final, desta forma ao fim deste tutorial teremos um software. Este software será um Media Player capaz de reproduzir arquivos de áudio, exibir lista de músicas, imagens de capa de CD obtidas da internet, visualizador de espectro e etc.

Com todo o conhecimento obtido neste tutorial será possível fazer um Media Player e ainda passível de explorações pois o leitor estará apto a gerar soluções de forma independente. Nos próximos capítulos serão demonstrados as características básicas de criação de interfaces gráficas.
