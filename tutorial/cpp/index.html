Prefácio

Esta introdução à programação em linguagem C++ foi desenvolvida com foco no ensino de orientação a objetos em cursos de engenharia. Tais cursos geralmente demandam apenas conhecimentos básicos da linguagem, dadas as limitações de tempo impostas nos currículos e a carga de outras disciplinas presentes nos currículos. Empestar de conteúdos mais avançados em programação a rotina de futuros engenheiros pode sobrecarregar a audiência e colocar o público para correr.

Visando deixar os alunos motivados, procuramos encher este material apenas com ferramentas simples e úteis que servirão para tocarem atividades acadêmicas e profissionais na área de engenharia.

Não espere se tornar um especialista em C++ após resolver todos os exercícios propostos. Apenas a prática constante lhe dará experiência para identificar as melhores estratégias de programação para resolver os problemas que poderão surgir.

Embora existam outras linguagens que também possam servir para o ensino de orientação objetos, a escolha pela linguagem C++ se dá por uma forte razão: problemas de engenharia que envolvem programação geralmente estão relacionados a aplicação de métodos numéricos ou interação com dispositivos eletro-eletrônicos.

Como estas duas searas normalmente envolvem a necessidade de computação performante ou interface com outras linguagens de baixo nível (por exemplo, Assembly), a linguagem C geralmente é a preferida nessas ocasiões. Logo, a escolha imediata da linguagem C++ para realizar interações de alto nível é bem atrativa para tais programadores.

Serão apresentados ao leitor conceitos rudimentares de orientação a objetos em C++, construtores e destrutores, alocação dinâmica de memória, herança, polimorfismo, classes abstratas, tratamento de exceções, sobrecarga de operadores e de funções.

Apresentamos também alguns recursos da biblioteca padrão de gabaritos STL (www.sgi.com/tech/stl), uma espécie de canivete suíco em algoritmos e estruturas de dados. Acreditamos que tais conceitos são razoáveis para alunos de engenharia que desejam se aventurar nos meandros da programação em C++.

Os exemplos apresentados no texto foram testados com sucesso em um sistema Linux.

1. Introdução

Desde que começaram a se espalhar, as linguagens de programação vêm acompanhando de perto a evolução do hardware em que operam.

As primeiras linguagens que surgiram por aí foram projetadas para dispositivos bem mais modestos que o seu telefone celular.

Em verdade, possuíam limitações de memória e velocidade nos sistemas da época. A arquitetura dos antigos processadores forçava os pioneiros a realizar verdadeiros malabarismos para implementar programas que hoje conseguimos desenvolver com grande facilidade em equipamentos baratos comprados na loja da esquina. Para leitores curiosos e com tempo livre, vale a pena gastar alguns minutos lendo sobre a história da programação e ver as dificuldades enfrentadas pelos nossos antepassados.

Se olharmos para os programas de computador criados no passado e compararmos com os criados no presente, não é muito difícil concluir que é preciso cada vez mais esforço para conseguir prosperar na área de programação de computadores.

Antigamente, a oferta de desenvolvedores de software e de ferramentas de desenvolvimento era bem mais escassa. Hoje, com a disseminação dos meios de educação em massa, de ferramentas de desenvolvimento gratuitas e de mecanismos eficientes para comercialização de programas de computador, está bem mais difícil encontrar compradores para programas do tipo "hello, world!".

As consequências naturais da evolução do mercado de programas de computador são o aumento de complexidade dos sistemas criados e do número de profissionais envolvidos no desenvolvimento das ferramentas.

O uso de tecnologias apropriadas para gerenciar o caos nestas situações é imprescindível para quem quer se enveredar na área. Alunos com formação específica em Ciências da Computação são adestrados desde cedo para aprender importantes conceitos de engenharia de software que os ajudam a organizar as partes de um sistema complexo, de modo a torná-lo gerenciável. Entretanto, cursos de engenharia possuem um foco bem diferenciado, mais voltado para o aprendizado da matemática, da física e da química.

Para este público, o aprendizado dos paradigmas de orientação a objeto e as funcionalidades de uma linguagem orientada a objetos pode ser um diferencial no mercado de trabalho. Grande parte das ferramentas de desenvolvimento profissionais oferecem amplos recursos de orientação a objetos, facilitando a modelagem e a análise de sistemas bem mais complexos que o belo e simples "hello, world!".

Existem diversas linguagens de programação capazes de realizar tarefas de seu interesse no computador. As linguagens de programação podem ser classificada de acordo com a forma como o programador especifica o problema a ser resolvido, ou seja, conforme o paradigma de programação suportado pela linguagem.

Provavelmente, o paradigma mais conhecido é o procedural (presente na linguagem de programação C), mas há outras opções também disponíveis por aí, capazes de permitir a solução de algumas classes de problemas de forma mais eficiente. Em todo caso, os paradigmas de programação mais comuns são os seguintes:

  • Procedural

  • Funcional

  • Lógico

  • Orientado a objeto

Cada um desses paradigmas será brevemente apresentado nas seções seguintes, ilustrando classes de problema cada um se aplica.

1.1. Paradigma de programação procedural

O paradigma de programação procedural geralmente é o preferido dos iniciantes. Foi um dos primeiros paradigmas de programação adotados, pois permitia (e ainda permite) que diversos algoritmos, principalmente aqueles que envolviam computação numérica, fossem criados com relativa facilidade.

Para o ensino da programação, o paradigma procedural facilita a introdução aos alunos dos conceitos básicos de concepção de algoritmos, de entrada e saída de dados e de tratamento de memória. Tais conceitos são normalmente tratados em cursos introdutórios de programação para alunos de engenharia e normalmente atendem às exigências curriculares da maioria dos cursos.

Programas de computador escritos em linguagens que seguem este paradigma geralmente são compostos por um conjunto de módulos (ou funções), cada um realizando uma determinada parte do processo. Cada programa é composto de uma rotina principal e diversas subrotinas auxiliares, sendo dada ênfase principal à ordem dos eventos no sistema desenvolvido, conforme ilustrado na Figura Paradigma de programação procedural. .

Figura 1. Paradigma de programação procedural

Normalmente, linguagens puramente procedurais oferecem um melhor controle do fluxo do código e são mais fáceis de entender.

Como o foco é dado na sequência de eventos, elas geralmente são bastante comuns nas comunidades que lidam com a concepção e implementação de algoritmos numéricos.

As bibliotecas mais difundidas para uso em computação numérica são escritas em linguagens procedurais como C ou FORTRAN.

1.2. Paradigma de programação orientada a objetos

Na programação orientada a objetos, um programa de computador é visto como um conjunto de agentes que colaboram de modo a realizar um problema.

Nesta filosofia de programação, a ênfase é dada forma como os agentes que causam as ações ou que estão sujeitos às ações de outros agentes.

Os programas de computador são formados por um conjunto de entidades (objetos) que trocam mensagens entre si.

A ideia de um programa desenvolvido sob a ótica do paradigma de programação orientada a objetos é ilustrada na Figura Paradigma de programação orientada a objetos.

As linguagens que suportam orientação a objetos oferecem uma grande vantagem para o programador: permitir a separação de dados e procedimentos de forma mais evidente.

Esta característica permite, entre outras coisas, que o programador desenvolva os módulos de sua aplicação atenuando a árdua tarefa de lidar com o tráfego de informação entre funções.

2. Ferramentas de desenvolvimento

2.1. Preparando o ecossistema de desenvolvimento

Como toda linguagem de programação, programas em C++ precisam ser editados de alguma forma. O código-fonte desenvolvido em C++ deve ser armazenado na forma de texto simples. A princípio, qualquer editor de texto pode ser usado para editar C++. Não há problemas em editar seus programas no Word ou BrOffice. Entretanto, a comodidade de usar um ambiente de desenvolvimento integrado para criar aplicações é incontestável. Ambientes integrados permitem, por exemplo, realçar com cores o código escrito, deixando mais agradável a tarefa de programação. Ambientes integrados são capazes de automatizar o processo de organização e compilação do código-fonte, focando a atenção do programador no desenvolvimento de algoritmos.

Sendo assim, o passo zero para o aprendizado da linguagem é preparar seu computador com as ferramentas necessárias para editar e compilar programas escritos em C++.

A propósito, C++ é uma linguagem compilada, de sorte que necessita de uma ferramenta para processar os arquivos que contém código C++ e gerar um programa executável. Outros tipos de linguagens podem ser interpretadas, necessitando de um programa que leia as instruções e as execute, sem gerar um programa executável (Ex: Python), ou meio lá, meio cá, um híbrido de compilação com interpretação, como é a proposta da linguagem Java. Entretanto, deixamos essas discussões para outras esferas no aprendizado de linguagens de programação.

Na qualidade de usuário de sistemas Unix, fico acanhado em sugerir um ambiente de programação descente para usuários Windows. Dos que já mantive contato, o Dev-C++ é uma opção interessante, embora acredite que devam existir outras mais atraentes.

Em ecossistemas Unix, por outro lado, existem várias opções de ambientes de desenvolvimento de código em C++ e todas elas geralmente são integradas ao compilador mais popular na comunidade livre: o GCC/G++. Nesta linha, podem ser sugeridas algumas opções de ambientes que possam interessar aos programadores: eclipse, netbeans, KDevelop e Qt Creator.

Não dá para afirmar qual deles é o melhor para escolher, pois geralmente a comodidade do usuário normalmente é o fator mais preponderante. O que muda basicamente de uma proposta para outra (do ponto de vista de um usuário iniciante) é o apelo visual e funcionalidades de integração com o sistema em que o usuário desenvolve. A maior parte das tarefas de edição de texto e código realizo usando editores ASCII como o emacs e compilo os arquivos gerados em um terminal através de linhas de comando. Quando há necessidade de desenvolver aplicativos com interface gráfica em C++, faço uso da biblioteca QT (http://qt.nokia.com) e utilizo o Qt Creator para escrever código e integrar comodamente toda a criação.

Programa simples desenvolvidos em C++ puro geralmente fazem uso de terminais para exibir os resultados produzidos, inclusive no sistema operacional Windows. 2.2. Descriçao do programa testa-ambiente.cpp

O programa inicia com uma diretiva de inclusão que insere para processamento o arquivo iostream.

A diretiva #include instrui o compilador a usar um pré-processador que lê e processa o arquivo referenciado antes de processar o restante do programa.

#include <iostream>

O arquivo iostream contém elementos importantes para a lide com a entrada e a saída de dados padrão. Entenda-se como entrada e saída padrão os dispositivos que realizam recepção de dados e exibição de resultados para o usuário. Na maioria dos sistemas são relacionados com o teclado e o terminal (monitor), respectivamente, onde um dado programa é executado.

Em seguida, aparece o chamado a mais uma diretiva de compilação: a diretiva using.

using namespace std;

Namespaces permitem ao usuário agrupar elementos de um programa sob a tutela de um nome (o namespace). Por exemplo, suponha que você tivesse a disposição duas funções com o protótipo float media(float,float), mas com comportamentos diferentes. Uma delas calcula a média aritmética, e a outra, a média geométrica. Para evitar reescrever o nome da função, o uso de namespaces poderia servir para discriminar entre uma e outra. Por exemplo, Geometrica::media(3,3) chamaria a função média no escopo do namespace Geometrica, e Aritmetica::media(3,3) chamaria a função média implementada no escopo do namespace Aritmetica. A diretiva serve para selecionar que namespace (ou subconjunto dele) deverá ser usado a partir da referida declaração. As funções da biblioteca padrão do C++ são todas declaradas no namespace std, daí a declaração estar presente na maioria dos códigos desenvolvidos.

Por enquanto, basta saber que o uso da chamada da declaração acima evita que os nomes definidos no namespace std possam ser chamados sem usar o prefixo std::. Diversos elementos importantes para o usuário estão definidos dentro desse namespace std, tais como os que permitem realizar entrada e saída de dados. Por isso, é bastante comum tal declaração ser necessária na maioria dos programas em C++.

A função main() vem definida logo em seguida. Assim como em C, todo programa em C++ também inicia pelo chamado dessa função. Sendo assim, apenas uma função main() é permitida por programa.

int main(void){
...
}

As operações de entrada e saída de dados em C++ são bastante diferentes do que se costuma usar na linguagem de programação C. Em C, as funções de entrada e saída são printf() e scanf(), respectivamente. Em C++, duas entidades são usadas para desempenhar esse papel: cin e cout. No exemplo, encontra-se a seguinte declaração:

cout << "Digite seu nome: ";

Quando executado, esse trecho de código imprime na saída o literal "Digite seu nome:". Sem querer adiantar assunto que será apresentado no futuro, inevitavelmente é necessário apresentar cout como o objeto pré-definido do namespace std responsável por receber variáveis, formatá-las e enviá-las para impressão. O símbolo que aparece logo em seguida indica para onde o literal da direita está sendo enviado. Neste caso, cout. Pelo fato de enviar elementos para cout, o símbolo é também conhecido como operador de inserção.

Embora o conceito de objeto só seja apresentado mais adiante, não há como apresentar os elementos que permitem ler e escrever sem tocar um mínimo no assunto. Sendo assim, fica registrado que cout é um objeto que serve como ponte para enviar coisas ao terminal de saída. E quanto à função printf()? Também pode ser usada? Sim! Ela é, inclusive, mais rápida para escrever que a solução usando cout. Entretanto, o uso do cout é menos propenso a erros, devendo ser preferido para programação em C++.

Finalmente, as declarações finais realizam a leitura de um literal digitado pelo usuário e reproduz o que foi lido na saída:

cin >> s;
cout << "Seu nome é " << s << endl;

O símbolo endl no final da declaração equivale a uma quebra de linha, surtindo efeito semelhante ao uso do "\n".

O objeto cin propõe uma funcionalidade semelhante à do cout, mas destinado à entrada de dados. Ele normalmente é conectado à entrada padrão (teclado) e as informações inseridas pelo usuários podem ser enviadas ao programa pelo operador de extração >>, que aparece logo após o cin.

A propósito, os objetos cin e cout podem lidar com mais de um dado na mesma linha de declaração, como ilustra a última linha do exemplo anterior.

3. Classes e objetos

3.1. Definindo classes e objetos

Dois conceitos principais destacam-se no tocante às linguagens orientadas a objeto, que são os de objeto e classe.

Um objeto do mundo real pode ser definido como uma entidade tangível que apresenta algum comportamento bem definido. Os objetos geralmente fazem alguma coisa e a interação com eles se dá através de mensagens que lhes são passadas. Entenda-se mensagem como alguma ação que foi requisitada para o objeto realizar. Por exemplo, para se mudar o canal de uma TV usando o controle remoto, aperta-se um botão e o controle envia um sinal eletromagnético que faz o canal mudar. A mensagem enviada para o controle, neste caso, foi a solicitação de mudança de canal, quando um determinado botão foi pressionado.

Um objeto possui estado, comportamento e identidade. O estado indica todas as propriedades de um objeto, bem como os valores dessas propriedades. O comportamento indica como um determinado objeto reage perante mudanças no seu estado e durante a passagem de mensagens. Finalmente, a identidade é a propriedade que diferencia um objeto de todos os outros. Para o caso do controle remoto, por exemplo, pode-se afirmar que seu estado poderia ser representado por diversas propriedades, como cor (preto, branco, amarelo), quantidade e tipos de botões, se os botões estão pressionados ou não, se ele está emitindo sinais ou não, etc.. O comportamento é definido pelas ações que controle remoto é capaz de realizar, ou seja, emitir um sinal eletromagnético diferente para cada um dos botões que é pressionado. A identidade do controle remoto é assegurada pela sua existência única, ou seja, embora possam existir outros exatamente idênticos, apenas do controle em questão ocupa o seu lugar no espaço.

Em se tratando de um objeto digital, existente somente no computador, as mesmas propriedades são válidas. Objetos digitais também possuem estado, comportamento e identidade. O estado é definido pelos valores que as variáveis presentes no objeto assumem, o comportamento é determinado pelas funcionalidades que os objetos implementam, e a identidade é assegurada pelo fato de apenas um dado objeto de cada vez ocupar um espaço determinado de memória. Os mecanismos para definir estes objetos serão vistos nas seções que se seguirão.

A classe, por sua vez, é um conjunto de objetos individuais que partilham características comuns e um comportamento comum. A classe dos mamíferos, por exemplo, descreve as características comuns de todos os animais de um determinado grupo. As características mais determinantes são o fato de mamarem ao nascer e possuirem pêlos no corpo. Sendo assim, o homem, que possui estas duas características, pertence à classe dos mamíferos. Em outras palavras, poderíamos afirmar que Um homem é um instância da classe dos mamíferos.

Os objetos de uma mesma classe podem até assumir o mesmo estado, mas não a mesma identidade. O que os diferencia é justo o fato de ocuparem diferentes lugares no espaço, ou na memória de uma máquina (no caso de objetos digitais).

Dessa forma, apresentamos a definição que relaciona os conceitos de classe e objeto:

Relação entre classe e objeto

Uma classe é um conjunto de objetos que partilham características comuns. Um objeto é uma instância de uma classe.

3.2. Abstraindo uma classe

Todo o desenvolvimento de classes e objetos em C++ é feito com base no conceito de abstração. Uma abstração é uma descrição resumida de um objeto que apresenta detalhes que são interessantes para seu usuário e esconde aqueles que não são. O processo de abstração permite organizar objetos complexos na forma de um conjunto de objetos cada vez menos complexos, de modo a permitir que se entenda cada pedaço desse sistema complexo de forma independente.

Dessa maneira, é mais fácil relacionar os objetos desse sistema e entender seu funcionamento como um todo. Para compilar um programa de computador, por exemplo, o usuário abstrai a máquina, ignorando todo o complicado processo de acionamento dos circuitos combinacionais que são necessário para gerar um simples programa executável.

Para programas de computador, a abstração permite que sistemas compostos de milhares de linhas de código sejam fracionados em porções menores. Fazendo assim, o programador é capaz de se concentrar apenas nas tarefas que cada porção realiza enquanto estiver realizando o processo de desenvolvimento.

Em C++, a abstração de uma classe é descrita na forma de um trecho de código que contém as informações essenciais para representar a estrutura que se deseja.

Considere, por exemplo, o conceito de vetor bidimensional. Tais vetores poderiam ser descritos, por exemplo, na forma de dois pares ordenados \$(x_0, y_0)\$ e \$(x_1,y_1)\$, denotando os seus pontos inicial e final, como mostra a Figura Modelo de vetor bidimensional.

Figura 3. Modelo de vetor bidimensional

Para simplificar o processo de abstração da classe vetor, vamos assumir que todos os vetores possuem seus pontos iniciais localizados na origem do sistema de coordenadas, ou seja \$(x_0, y_0) = (0,0)\$. Dessa forma, um vetor existente no plano seria representado apenas pelas coordenadas do seu ponto final, definindo, assim, sua orientação e sentido.

Assumindo também que a comunicação com os objetos da classe vetor se dá através de mensagens que lhes são repassadas, é razoável assumir para seu comportamento funcionalidades bastantes para definir os valores das coordenadas do ponto final do vetor, bem como para recuperar o estado do objeto, lendo os valores dessas coordenadas.

Neste ponto, levantamos uma questão importante: se o estado do objeto está representado pelos valores assumidos pelas suas variáveis, por que não atribuir ou recuperar valores diretamente dessas variáveis, ao invés de criar funcionalidades só para isso? Para poder encapsular e esconder a informação! Pelo encapsulamento, dados e funções de uma classe são representados em uma mesma estrutura cujo principal objetivo é não permitir que o programador realize acesso direto aos dados.

Encapsulamento

É o processo de organizar os elementos de uma abstração de modo que os clientes de uma determinada classe saibam apenas o necessário sobre o comportamento dos objetos que a ela pertencem.

Entenda por cliente de uma classe quais quer estruturas (classes ou funções) que usam objetos dessa classe para realizar alguma operação. Na Listagem Código para teste de ambiente, por exemplo, a função main() é cliente da classe std::iostream, pois usa os objetos cin e cout, que estão nela definidos.

A definição de um classe em C++ basicamente cria um novo tipo de dado, semelhante à uma struct da linguagem C. A diferença básica é que funções que representam o comportamento do tipo também podem ser incluídas.

O processo de encapsulamento permite que o usuário tenha controle sobre o que pode e o que não pode ser observado ou modificado nas entranhas dos objetos de uma classe. Isto é feito através do uso de especificadores de acesso que, em C++, são três:

Public (públicas)

Declarações que são acessíveis por todos os clientes.

Private (privadas)

Declarações que são acessíveis somente pela classe e suas amigas (friends).

Protected (protegidas)

Declarações que são acessíveis somente pela classe, pelas suas amigas e pelas suas subclasses.

Por enquanto, não é interessante entrar em detalhes acerca do que são amigas e subclasses. Vamos apresentar primeiro os conceitos de acesso público e privado usando um exemplo simples, para não sobrecarregar a compreensão de conceitos apresentados.

Vamos apresentar uma possível implementação para a classe dos vetores bidimensionais. Antes, porém, é importante refletir um pouco sobre como representar um vetor. Dessa forma, estruturas adequadas serão escolhidas para resolver o problema e abstrair do usuário da classe o funcionamento interno dessa.

A implementação mínima para uma classe de vetores bidimensionais deve prover mecanismos para armazenar pelo menos duas informações importantes, que são os valores dessas coordenadas \$x\$ e \$y\$. Variáveis do tipo float podem ser usadas para esse fim.

Ainda, devem existir funções que façam as seguintes tarefas:

  • Definam os valores das coordenadas \$x\$ e \$y\$ do vetor.

  • Recuperem os valores das coordenadas \$x\$ e \$y\$ do vetor.

3.3. Revisitando a definição de classes

Qualquer função definida dentro da declaração da classe torna-se, automaticamente, uma função inline (em linha), ou seja, o código criado é copiado diretamente para o trecho de programa onde o objeto é utilizado. Os benefícios de funções inline são traduzidos na forma melhoria de performance para o programa. Em contrapartida, os programas demandam mais memória para executar, o que pode ser problemático em sistemas com limitação de recursos. Sendo assim, o uso de funções inline deve ser bem dosado, com preferência para funções membro curtas.

O processo de desenvolver uma classe geralmente demanda um conjunto de passos, ao invés de criar tudo de uma vez só. O programador deve fragmentar seu programa em um conjunto de partes menores, visando tornar o código mais compartimentalizado, separando a definição da classe da sua implementação.

Considere a Listagem Classe Vetor2d que trata da representação de vetores bidimensionais. Seguindo a ideia da compartimentalização do código, poderiamos separar o arquivo de código gerado em três arquivos relativamente independentes: vetor2d.hpp (Listagem Classe Vetor2d), contendo a definição da classe Vetor2d, vetor2d.cpp (Listagem vetor2d.cpp), contendo a implementação da classe Vetor2d, e main.cpp (main-vetor2d.cpp), o programa cliente que faz uso da classe criada.

4. Construtores e destrutores

Uma questão importante que pode despertar a atenção do programador é como estabelecer o estado de um determinado objeto assim que este é criado. Considere, por exemplo, a classe Vetor2d que vem sendo estudada ao longo do texto. Logo que um objeto dessa classe é declarado, que valores assumem as variáveis x e y inicialmente? A priori, pode-se afirmar que são os conteúdos presentes nas regiões de memória reservadas para abrigar estas duas variáveis. Geralmente, é praticamente impossível prever que valores seriam estes. Desse modo, a existência de mecanismos que inicializem as variáveis internas do objeto para que seu estado inicial seja previamente conhecido pode ser bastante conveniente.

Uma solução seria, por exemplo, criar um método iniciar() que atribuísse às variáveis valores conhecidos (por exemplo, \$x=0\$ e \$y=0\$) e disponibilizá-lo na interface pública da classe. Sendo assim, cada vez que um objeto da classe fosse criado, o usuário poderia acionar o método, inicializando o objeto, como sugere o exemplo a seguir.

Vetor2d v1;
v1.iniciar();

Entretanto, existem recursos mais elegantes disponíveis para os programadores: são os chamados construtores. Os construtores são funções especiais padronizadas que podem ser adicionadas a uma classe. Quando existentes, elas são chamadas automaticamente exatamente após o objeto ser criado, permitindo que o programador defina todos os procedimentos necessários para inicializar o estado do respectivo objeto após a sua criação. Em outras palavras, eles permitem construir o estado inicial de um objeto.

Em C++, os construtores são funções que recebem o mesmo nome da classe, podem existir mais de um por classe, e são diferenciados pelo número e tipos de argumentos. Por exemplo, para a classe Vetor2d, um dos possíveis construtores é uma função membro chamada Vetor2d().

Suponha que, quando criados da forma mostrada no exemplo, objetos da classe Vetor2d devam assumir em seu estado inicial os valores \$(x,y)=(0,0)\$ para as variáveis da classe. Para assegurar este comportamento, o construtor da classe deve prever esta funcionalidade, por exemplo, com a definição mostrada na Listagem Exemplo de definição de um construtor.

Listagem 6. Exemplo de definição de um construtor
Vetor2d::Vetor2d(){
  x=0;
  y=0;
}

Observe que, como outras funções da classe, o construtor também possui acesso direto aos membros privados da classe, permitindo a modificação de seus valores.

Embora atribuir automaticamente valores nulos para as coordenadas do vetor seja bastante conveniente, o cliente da classe pode desejar escolher seus próprios valores, de sorte que o construtor proposto torna-se insuficiente. Para resolver este problema, um novo construtor pode ser criado para receber através de dois argumentos passados pelo usuário os valores desejados para as coordenadas do vetor. Uma possível definição para o novo construtor seria a seguinte:

Vetor2d::Vetor2d(float _x=0, float _y=0){
  x=_x;
  y=_y;
}

O novo construtor agora recebe as coordenadas pelas variáveis _x e _y, que são copiadas para as variáveis privadas da classe. Veja que os nomes das variáveis não são so mesmos declarados na classe, para evitar confusões para o compilador. O caractere underline pode ser utilizado no início de cada variável que receberá os parâmetros para evitar a ambiguidade, e ainda identificar a que variável se deseja atribuir o valor repassado.

Veja ainda que os parâmetros recebem argumentos default. O argumento default é um recurso da linguagem C++ que permite atribuir valores automaticamente caso seja omitido o argumento correspondente para a função chamada. Argumentos default devem ser fornecidos da direita para a esquerda de modo que não se pode prover valores para um argumento default a menos que todos os argumentos à sua direita tenham valores default também atribuídos. Exemplo:

Vetor2d::Vetor2d(float _x=0, float _y=0); // VÁLIDO
Vetor2d::Vetor2d(float _x, float _y=0); // VÁLIDO
Vetor2d::Vetor2d(float _x=0, float _y);   // INVÁLIDO

Assim, a classe Vetor2d pode ser redefinida da seguinte da seguinte forma usando um construtor como mostra o exemplo a seguir. Procure recompilar o código anterior, incluindo modificações para testar o construtor e verifique os efeitos produzidos.

#ifndef VETOR2D_HPP
#define VETOR2D_HPP

class Vetor2d{
private:
  float x, y;
public:
  Vetor2d(float _x=0, float _y=0){
    x=_x; y=_y;
  }
  void setX(float _x);
  float getX(void);
  void setY(float _y);
  float getY(void);
};

#endif

A criação de um objeto sempre é feita pela invocação de seus métodos construtores. Sempre que uma função membro possui o mesmo nome da classe, ela é considerada um construtor. Três tipos de construtores podem ser identificados em um programa escrito em C++: o construtor default , o construtor de cópia, e o construtor com argumentos variados. Quando um construtor não é fornecido pelo programador, o próprio compilador oferece uma implementação, que é inserida automaticamente no código gerado.

4.1. Construtor default

O primeiro deles é um construtor sem argumentos, que é chamado toda vez que um objeto é criado sem argumentos. Um construtor default normalmente é disponibilizados pelo compilador, na ausência de uma implementação fornecida pelo usuário. Normalmente, os construtores desse tipo não possuem argumentos. Entretanto, um construtor onde todos os argumentos recebem valores default também é considerado um construtor default. Pode haver somente um desses construtores para cada classe, pois a inclusão de mais de um causa confusão durante o processo de escolha pelo compilador. Alguns exemplos de construtores são mostrados no Exemplos de construtores default.

Listagem 7. Exemplos de construtores default
class Alo{
  int x;
  public:
  Alo(){ x=0; }
};

int main(void){
  Alo x;
}


class Alo{
  int x;
  public:
  Alo(int _x=0){ x=_x;}
};

int main(void){
  Alo x;
}

4.2. Construtor de cópia

O segundo tipo de construtor é o construtor de cópia. Ele é usado quando se deseja criar um novo objeto com as características de um outro que é passado como referência na forma de um parâmetro. Considere o exemplo a seguir.

#include <iostream>

using namespace std;

class Alo{
  int x;
public:
  Alo(int _x=0){
    cout << "construtor default\n";
    x= _x;
    print();
  }
  Alo(Alo &a){
    cout << "construtor copia\n";
    x = a.x;
    print();
  }
  void print(){
    cout << "x = " << x << "\n";
  }
};

int main(void){
  Alo x;
  Alo y(1);
  Alo z = x;
  Alo w = y;
  Alo k(y); // mesmo que Alo k = y; !
}

Quando compilado e executado, o exemplo acima produz a seguinte saída:

construtor default
x = 0
construtor default
x = 1
construtor copia
x = 0
construtor copia
x = 1
construtor copia
x = 1

As duas primeiras instanciações da classe Alo chamam diretamente o construtor default . Na primeira instanciação a chamada do construtor não recebe argumentos, de sorte que o valor default para a variável x é assumido. Na segunda, um valor inteiro é repassado, que por sua vez é armazenado na variável.

Na terceira instanciação da classe Alo, a variável z é criada. Desta vez, a atribuição de igualdade à variável x faz com que o segundo construtor seja chamado. Neste construtor, é possível observar uma nova forma de passagem de argumentos: a passagem por referência. A passagem de argumentos por referência é identificada pelo uso do símbolo & precedendo o nome da variável.

Observe a Figura Passagem de parâmetros por referência. A variável y pertencente à função main() assume o valor 3, que é armazenado em uma posição de memória destinada para ela. Entretanto, sob a ótica da função funcao() esta mesma posição de memória onde o inteiro 3 está armazenado é vista como sendo a variável x. Dessa maneira, uma variável é declarada em um local e referenciada em outra usando um nome diferente.

Figura 5. Passagem de parâmetros por referência

Embora a sintaxe das referências possa ser mais simples e transparente que a sintaxe de ponteiros, nem sempre podem ser usadas. Por exemplo, referências não podem receber outros valores uma vez que são inicializadas (ponteiros podem); as referências nunca podem ser nulas, embora os ponteiros possam ser. Via de regra, deve-se preferir o uso de referências, escolhendo o uso de ponteiros somente se necessário for.

Construtores de cópia geralmente são escolhidos quando se precisa alocar recursos disponíveis além da definição da própria classe, por exemplo, em operações que envolvem alocação dinâmica de memória, manipulação de arquivos e acesso a recursos do sistema operacional. Assim, além de realizar a cópia direta de variáveis alocadas estaticamente, o construtor de cópia deverá gerenciar os problemas de alocação de recursos para o novo objeto.

Construtores de cópia possuem ainda uma classificação adicional. Podem ser chamados de construtores de cópia rasa ou profunda, conforme os procedimentos de alocação de recursos que realizam. Por exemplo, se apenas a cópia membro-a-membro de um objeto para outro é realizada, diz-se que é feita uma cópia rasa do objeto. Entretanto, se recursos adicionais são alocados, então uma cópia profunda é realizada.

4.3. Construtor com argumentos variados

Quaisquer outros construtores que não se enquadrem nas duas categorias previamente descritas podem são construtores com argumentos variados. Neste caso, fica a cargo do programador especificar quantidade e tipos de argumentos que serão fornecidos ao construtor.

Listagem 8. Exemplos de construtores com argumentos variados
class Alo{
  int x;
  public:
  Alo(float z, int w){}
};

int main(void){
  Alo x(5.8,-3);
}

É importante lembrar que o que difere um construtor de outro são os seus argumentos, uma vez que o nome da função construtor é exatamente a mesma. Sendo assim, deve-se tomar cuidado para evitar declarações que causem confusão para o compilador, como mostra o exemplo seguinte:

class Alo{
  int x;
public:
  Alo(int _x=0){ x=_x; } // Problema:
  Alo(){ x=0; }          // qual dos dois deverá ser chamado?!
};

int main(void){
  Alo x;
}

A sobrecarga (presença de mais de uma implementação) dos dois construtores é ambígua, de sorte que o compilador apresenta erro durante o processamento do código.

4.4. Destrutores

Da mesma forma que os construtores, os destrutores também desempenham um processo importante na linguagem C++. Eles são chamados automaticamente quando o objeto está para cessar de existir na memória da máquina. Portanto, são considerados um recurso valioso para realizar liberação de recursos alocados pelos construtores ou acionar outros mecanismos necessários durante a fase de destruição de um objeto criado.

Ao contrário dos construtores, que podem existir mais de um, somente um destrutor pode existir, de modo que é vedado ao programador escolher a função de destruição, que é única e não recebe parâmetros. Ela recebe o mesmo nome da classe precedida pelo símbolo ~, como ilustra a Listagem Exemplo de definição de um destrutor.

Listagem 9. Exemplo de definição de um destrutor
#include <iostream>

using namespace std;

class Alo{
public:
  Alo(){
    cout << "Construtor de Alo\n";
  }
  ~Alo(){
    cout << "Destrutor de Alo\n";
  }
};

int main(void){
  Alo x, y;
}

Note que na sequência de chamadas de função, primeiro ocorre a do construtor e, em seguida, a do destrutor. Para cada objeto, as chamadas sempre ocorrem aos pares, de sorte que seu ciclo de vida sempre envolve a chamada de um construtor na criação e de um destrutor antes do seu fim. Caso o usuário não tenha incluido a função do destrutor na sua implementação, o próprio compilador se encarrega da tarefa, inserindo uma função vazia.

Considere agora um exemplo um pouco mais complexo envolvendo destrutores:

#include <iostream>

  }
  ~Alo(){
    cout << "Destrutor de Alo: elemento " << id << "\n";
  }
};

int main(void){
  Alo x(1), y(2), z(3);
}

Agora, quando mais de um objeto é criado, a sequência de chamadas de construtores e destrutores evidencia mais uma propriedade importante: a ordem em que os destrutores são chamados. O C++ instaura o processo de destruição automática primeiro para os objetos criados mais recentementes, partindo então para os objetos com criação mais antiga. Veja no exemplo acima que a criação de objetos é feita na sequência x → y → z, ao passo que a destruição segue a sequência z → y → x.

5. Herança

Entre os seres vivos, o mecanismo de herança permite que um filho receba características de seus pais, tornando-se semelhante as estes em praticamente todas as suas funcionalidades. Entretanto, os geralmente filhos apresentam aparência e comportamento diferentes dos pais, sinal de que sofreram algum tipo de especialização.

No processo de herança é estabelecido uma relação hierárquica entre as classes. A classe que possui as funcionalidades mais genéricas é geralmente chamada de classe base, ou classe herdada, enquanto que a classe que especializa o comportamento da classe base é chamada de classe derivada, ou classe herdeira.

Compreender adequadamente a relação estabelecida entre duas classes, base e herdeira, é importante para que se possa entender como as funcionalidades de uma classe são absorvidas, extendidas ou redefinidas por outra.

5.1. Efeitos do processo de herança

Todas as variáveis e funções presentes na classe base são recebidas e podem ser usadas pelas funções de uma classe herdeira, desde que observadas algumas restrições.

A principal delas diz respeito aos construtores e destrutores. Esses não são herdados da classe base, de sorte que o programador precisa desenvolver seus próprios métodos para a classe derivada.

Outra restrição à forma como é estabelecido o processo de herança. Observe no exemplo da Listagem Declaração da classe Motor usando herança a palavra reservada public que aparece na linha que cria a relação de herança entre as classes Equipamento e Motor. . Este especificador de herança determina que se uma função ou variável com visibilidade pública (declarada após ) na classe base esta será vista também como pública na classe herdeira. Da mesma forma, uma função ou variável com visibilidade protegida (declarada após o especificador ) também será protegida na classe derivada. Em outras palavras, a herança tipo public mantém as mesmas visibilidades para os membros públicos e protegidos da classe derivada.

Heranças do tipo private e protected também são permitidas, mas elas alteram o tipo de visibilidade dos membros públicos e protegidos da classe base. Na herança tipo private, todos os membros public e protected da classe base tornam-se private na classe derivada. Na herança tipo protected, todos os membros public e protected da classe base tornam-se protected na classe derivada.

5.2. Herança nos construtores e destrutores

O processo de invocação dos construtores e destrutores quando se usa herança é feito de forma cuidadosa de modo que os recursos necessários sejam alocados e desalocados na ordem correta.

A construção do objeto tem início na classe base e só então segue para a classe derivada, sendo regida pelas regras:

  • O construtor da classe base SEMPRE é chamado antes do construtor da classe derivada.

  • O construtor default é chamado automaticamente na classe base

  • Caso o construtor default não exista ou não faça sentido sua existência, deverá ser especificado explicitamente o construtor da classe base que deverá ser invocado. Se o usuário não informar o construtor adequado para ser chamado e o construtor default não existir na classe base, erros de compilação deverão surgir.

5.3. Atribuições de valores entre classes

Outro aspecto da herança na relação entre classes diz respeito ao processo de atribuição de valores entre classes criadas por herança. Quando se atribui a um objeto de uma classe um outro objeto, algumas coisas devem ser levadas em consideração.

Uma delas é o fato de objetos de uma classe base poderem receber atribuições de objetos de uma classe derivada, como mostra o seguinte exemplo:

class Base{
protected:
  int a, b;
public:
  void foo(int x, int y){
    a = x; b = y;
  }
  int getA(void) { return a; }
  int getB(void) { return b; }
};

class Derivada : public Base{
  int c;
public:
  void bar(int z){
    c = z;
  }
};

int main(void){
  Base b;
  Derivada d;
  b = d;
  d = b; // Erro! "d" possui parte indefinida.
}

Quando a atribuição b = d é realizada, o processo de cópia descarta a parte especializada do objeto d, fazendo a cópia das variáveis, membro a membro, no objeto b apenas da parte referente à classe Base. Já o processo inverso não é permitido. Quando uma atribuição do tipo d = b é realizada, parte do objeto d é indefinida. Sendo assim, um erro de compilação é gerado informando o problema.

Caso atribuição de uma classe base para uma classe derivada seja necessária, tal comportamento pode ser definido realizando sobrecarga de um operador de atribuição (operador =).

Observe o exemplo da Listagem [exe.heranca.atribuicoes.operator].

class Base{
  int a, b;
public:
  void foo(int x, int y){
    a = x; b = y;
  }
};

class Derivada : public Base{
  int c;
public:
  Derivada& operator=(Base &x){
    a = x.getA();
    b = x.getB();
    c = 0;
  }
  void bar(int z){
    c = z;
  }
};

int main(void){
  Base b;
  Derivada d;
  b = d;
  d = b; //OK! Parte indefinida de "d" agora eh tratada
}

A sobrecarga do operador de atribuição orienta a classe como tratar adequadamente membros de sua classe base destinados à cópia. Neste exemplo, o estado do objeto da classe Derivada é estabelecido pela observação cuidadosa dos membros da classe Base.

6. Sobrecarga de operadores

Em C, os operadores são símbolos especiais usados para calcular espressões matemáticas ou lógicas entre tipos de dados, podendo ser agrupados em duas categorias: binários e unários. Um operador é dito binário quando a operação que realiza envolve sempre dois operandos. Um operador é dito unário quando apenas um operando é envolvido.

Considere, por exemplo, o seguinte trecho de código:

int a, b, c;
a = 1; b = 2;
c = a + b;
c++;

A linha 3 mostra o uso de operador aritmético binário + para somar duas quantidades. Na linha 4, por sua vez, apenas um operando é envolvido e é, neste caso, processado pelo operador unário ++ para incrementar o valor da variável c. Como os tipos envolvidos no exemplo são tipos primitivos da linguagem usados para armazenar números inteiros comuns, as operações realizadas nestas linhas são óbvias para o leitor.

Considere agora este outro exemplo de uso de operadores:

class Foo{
  float x;
  string nome;
};

main(){
  Foo a, b, c;
  c = a + b;
}

Para este novo exemplo, a aplicação do operador + já não parece tão óbvia. Além disso, nem sequer é possível utilizar este símbolo, uma vez que não é previsto na linguagem C++ o uso deste operador para adicionar variáveis do tipo Foo. Um erro de compilação deverá ser apresentado! Neste caso, alguma orientação adicional deve ser provisionada para realizar a soma dos dois objetos da classe Foo que aparecem no exemplo.

Em C++, esta tarefa é realizada através de um processo de sobrecarga de operadores.

6.1. Criando a sobrecarga

A sobrecarga de operadores é realizada através de funções que dão novo sentido ao uso do operador. As funções devem ser nomeadas com a palavra reservada seguida do símbolo correspondente ao operador que se deseja sobrecarregar, como mostra a Listagem Realizando a sobrecarga do operador.

Listagem 17. Realizando a sobrecarga do operador
#include <iostream>
#include <string>
using namespace std;

class Foo{
  float x;
  string nome;
public:
  Foo(string _nome, float _x){
	nome = _nome;
	x = _x;
  }
  Foo(){}
  Foo operator+(Foo b){
	Foo retorno;
	retorno.x = x + b.x;
	retorno.nome= nome + b.nome;
	return(retorno);
  }
  void print(){
	cout << nome << ":"  << x << endl;
  }
};

main(){
  Foo a("jose",3), b("maria",4), c;
  c = a + b;
  c.print();
}

A função operator+() apresentada neste exemplo implementa a sobrecarga do operador. Nela, define-se o que fazer quando a instrução em C++ envolve dois operandos, ambos da classe Foo. Neste caso, resolveu-se criar um novo objeto dessa classe, para o qual o campo nome receberia a concatenação dos campos nome e o campo x assumiria a soma dos valores similares nos campos x dos dois operandos.

A instrução associada à soma de dois objetos da classe Foo foi denotada pela expressão

c = a + b;

mas poderia muito bem ser escrita da forma

c = a.operator+(b);

utilizando a chamada da função de sobrecarga pelo seu nome completo. Ambas são equivalentes. Entretanto, esta segunda alternativa é bem mais inconveniente e foge à simplicidade que se encontra disponível ao programador.

x = 1;

x = 1;
y = x++; // y = 1

Vetor2d Vetor2d::operator++(int){

Vetor2d Vetor2d::operator++(){

  return(Vetor2d(++x, ++y));

7. Ponteiros e alocação dinâmica de memória

Nas linguagens de programação, as variáveis são entendidas como blocos de memória capazes de armazenar valores de um determinado tipo. Por exemplo, se uma declaração da forma

int x;

é realizada, associa-se ao símbolo x o conceito de variável inteira, ou seja, x denota uma célula de memória capaz de armazenar um valor inteiro, desde que contido dentro de uma faixa de valores predefinida.

Variáveis primitivas como o inteiro x citadas no parágrafo anterior geralmente são usadas para a lide com dados comuns utilizados em um programa, como por exemplo para armazenar um contador de ocorrências, o número de rodas de um veículo ou a quantidade de registros em um banco de dados.

Entretanto, existe um outro tipo de variável em C/C++ concebida especificamente para armazenar endereços de memória: a variável ponteiro.

Diz-se que os ponteiros apontam para o endereço de memória que eles armazenam. Diversas operações em C++ somente são possíveis com o uso de ponteiros, sendo as mais comuns relacionadas à passagem de parâmetros para funções e à alocação dinâmica de memória.

O uso de ponteiros em C++ não difere muito do uso na linguagem C. Entretanto, em se tratando da aloçação dinâmica de memória, novos recursos são introduzidos de modo a contemplar a sistemática das classes e objetos da linguagem.

  void setY(float _y){
	y = _y;
  }
  float getY(void){
	return y;
  }
};

int main(void){
  Vetor2d v1;
  v1.setX(4.5);
  v1.setY(-1.3);
  cout << v1.getX() << " " << v1.getY() << "\n";
}

Repare que o tipo de dado float foi escolhido para representar as coordenadas espaciais \$(x,y)\$ para os objetos da classe.

O problema se agrava ainda mais quando se faz necessário dispor de diversas implementações semelhantes à classe Vetor2d para outros tipos de dados, como int, long, double, char etc.. Esta solução é, de fato, impraticável e formas alternativas de solucionar este problema encontram-se disponíveis.

Em C++, a solução é feita usando programação genérica com gabaritos (ou templates ).

Um gabarito é um tipo especial de definição que permite ao usuário criar funções e classes adaptadas para um ou mais tipos sem a necessidade de reescrever todo o código para cada tipo. Para isso, a programação com gabaritos permite que o usuário passe como parâmetro para a criação de determinadas estruturas os tipos de dados a serem usados, ao invés dos valores como é tradicionalmente realizado em C.

Dois tipos de gabaritos são previstos em C++: gabaritos de função e gabaritos de classe, destinados à programação genérica de classes e funções, respectivamente.

8.1. Gabaritos de funções ==

Um gabarito de função pode ser declarado usando a palavra reservada template, seguido da identificação dos tipos que se deseja utilizar e da declaração da função que sofrerá a ação do gabarito.

template <class identificador_do_tipo> declaracao_da_funcao;
template <typename identificador_do_tipo> declaracao_da_funcao;

O identificador do tipo é um símbolo que poderá aparecer na função que se segue. Todas as ocorrências deste identificador serão substituídas pelo tipo de dado fornecido pelo usuário na chamada da função. Na prática, não há diferença entre esses tipo de declaração além das palavras reservadas class e typename que aparecem. O efeito resultante do uso de uma ou outra forma é exatamente igual.

Cada declaração de gabarito influencia apenas o bloco funcional que lhe é imediatamente posterior, não surtindo efeito para os demais blocos funcionais que se seguem no programa.

A chamada de uma função criada com gabaritos, por sua vez, é feita incluindo na declaração o tipo que se deseja utilizar na função implementada e deverá seguir o seguinte molde:

funcao <tipo> (argumentos);

Para ilustrar o uso do gabarito, considere uma função que calcula a média entre dois números dados. Uma possível implementação para esta função é mostrada no Gabarito de função para cálculo de média aritmética .

Listagem 22. Gabarito de função para cálculo de média aritmética
#include <iostream>

using namespace std;

template <class Tipo>
Tipo media(Tipo a, Tipo b){
  Tipo resultado;
  resultado = (a+b)/2;
  return (resultado);
}

int main(void){
  int ix, iy;
  float fx, fy;
  double dx, dy;
  ix = 3; iy = 4;
  fx = 3; fy = 4;
  dx = 3; dy = 4;
  cout << media<int>(ix,iy) << "\n";
  cout << media<float>(fx,fy) << "\n";
  cout << media<double>(dx,dy) << "\n";
}

A primeira linha que precede a declaração da função media() caracteriza o início da definição do template.

template <class Tipo>

Este template deverá influenciar apenas o bloco funcional que lhe é imediatamente posterior. Neste caso, a implementação da função media(). O identificador Tipo que aparece dentro da definição do gabarito será substituido na função que se segue pelo tipo de dado fornecido pelo usuário quando chamar a função criada, como feito no exemplo:

media<int>(ix,iy)

Neste exemplo, quando o tipo int é passado como parâmetro na chamada da função média, todas as ocorrências do identificador Tipo serão substituídas pelo tipo inteiro fornecido. Quando o compilador encontra uma chamada para o template, ele automaticamente gera código para a função com a substituição mencionada e então chama esta função criada de modo totalmente invisível para o programador. Na prática, a função média é usada neste momento como se houvesse sido declarada da seguinte forma:

int media(int a, int b){
  int resultado;
  resultado = (a+b)/2;
  return (resultado);
}

Esta solução simples gera uma economia razoável de esforços de programação quando ocorre a necessidade de implementar funções que lidam com argumentos de tipos variados mas realizam exatamente as mesmas operações.

8.2. Gabaritos de classes

Gabaritos de classes operam de forma bem semelhante aos gabaritos de funções, seguindo os mesmos moldes na definição.

Usando o conceito de gabaritos é possível reescrever a classe Vetor2d apresentada no início deste capítulo de modo a torná-la genérica, adaptável a quaisquer tipos válidos que se deseje utilizar. Quando se menciona a expressão "tipos válidos" deseja-se chamar a atenção ao fato de que os tipos fornecidos para instanciar a classe devem prever todas as operações implementadas dentro dessa classe. Por exemplo, se na implementação da classe, um dos operadores da linguagem é aplicado ao tipo fornecido, este tipo (seja ele primitivo ou uma classe definida pelo usuário) deve prever como será feita a lide com este operador! A mesma restrição vale também para os gabaritos de função. O exemplo da Listagem Vetores com gabaritos apresenta a redefinição da classe Vetor2d usando gabaritos.

Redefinição da classe Vetor2d usando templates(arquivo

gabarito-vetor.cpp).

#include <iostream>

using namespace std;

template <class T>
class Vetor2d{
private:
  T x, y;
public:
  void setX(T _x){
	x = _x;
  }
  T getX(void){
	return x;
  }
  void setY(T _y);
  T getY(void);
};

template <class T>
void Vetor2d<T>::setY(T _y){
  y = _y;
}

template <class T>
T Vetor2d<T>::getY(void){
  return y;
}

int main(void){
  Vetor2d<float> v1;
  Vetor2d<int> v2;
  v1.setX(4.5);  v1.setY(-1.3);
  v2.setX(4.5);  v2.setY(-1.3);
  cout << v1.getX() << " " << v1.getY() << "\n";
  cout << v2.getX() << " " << v2.getY() << "\n";
}

Quando compilado e executado, a Listagem Vetores com gabaritos produz o seguinte resultado:

$ g++ gabarito-vetor.cpp -o gabarito-vetor
$ ./gabarito-vetor
4.5 -1.3
4 3

Neste exemplo, o símbolo T recebe o tipo fornecido pelo usuário e todas as ocorrências deste símbolo na implementação da classe são substituídas, durante a compilação, por este tipo. Assim como para os gabaritos de funções, o compilador gera automaticamente uma implementação da classe com o símbolo fornecido e então instancia propriamente o objeto. Na prática, nenhum código é gerado para uma classe ou função definida com gabaritos até que uma delas seja instanciada.

Na função principal, duas instanciações da classe são feitas: uma para o tipo int, associada ao objeto v1 e outra para o tipo float, associada ao objeto v2. Observe que os valores do par de parâmetros x e y passados para as funções setX() e setY() para ambos os objetos são exatamente iguais, mas produzem saídas diferentes. Ocorre que, como a classe relativa ao objeto v2 é gabaritada com o tipo inteiro, os valores passados para as funções set sofrerão conversão de tipo antes de ser armazenados, daí o truncamento da parte fracionária do número.

Note que dois dos quatro métodos da classe, setY() e getY() foram propositalmente implementados fora da declaração dessa classe. Neste caso, deve-se sempre incluir uma nova declaração de template antes de proceder com a implementação do método. Desta forma, linhas de código semelhantes às que se seguem deverão se repetir em todas as situações similares.

 template <class T>
T Vetor2d<T>::getY(void)

Neste trecho, o primeiro identificador T denota o nome da variável, o segundo identificador representa o tipo de retorno, e o terceiro, inserido entre os sinais < e >, indica que a instância da classe criada é aquela que foi gabaritada com o tipo fornecido.

Uma ressalva interessante cabe ao uso de gabaritos quando se deseja trabalhar com a programação em múltipos arquivos, ou seja, com a separação do código em um arquivo de cabeçalho (.h) e um arquivo de implementação (.cpp). A criação de classes e funções pelo uso de gabaritos é feita exclusivamente sob demanda, ou seja, tais estruturas somente passam a existir a partir do momento em que são instanciadas. Dessa forma, não há sentido algum em compilar em um arquivo separado uma classe definida através de gabaritos. Se assim fosse, por exemplo, na classe Vetor2d mostrada no exemplo, quantos bytes deveriam ser reservados pelo compilador para o tipo de dado T que é previsto na definição?

#include <iostream>

class Vetor2d{
  float x, y;
public:
  Vetor2d(float _x=0, float _y=0);

    x=_x; y=_y;
}

  return os;
}

istream& operator>>(istream& is, Vetor2d &v){

  return is;
}
