Clipping (recorte)

  • Processo de determinar que partes de uma cena devero ou no aparecer dentro de uma regio a ser exibida
  • Pode ser bidimensional ou tridimensional

Algoritmo de visibilidade simples

  • Um ponto $(x,y)$ encontra-se na regio de recorte se $x_L \le x \le x_R$ e $y_B \le y \le y_T$.
  • As linhas so ditas interiores regio se ambos os pontos finais forem interiores a essa. 
enum{TRUE, FALSE, PARTIAL};
foreach(linha){
  // a e b : pontos finais
  visibilidade = TRUE;
  if (xa < xL && xb < xL) visibilidade = FALSE;
  if (xa > xR && xb > xR) visibilidade = FALSE;
  if (ya < yB && yb < yB) visibilidade = FALSE;
  if (ya > xT && yb > yT) visibilidade = FALSE;
  if (visibilidade != FALSE){

        if (xb < xL || xb > xR) visibilidade = PARTIAL;
        if (ya < yB || ya > yT) visibilidade = PARTIAL;
        if (yb < yB || yb > yT) visibilidade = PARTIAL;
  }
  if (visibilidade == PARTIAL){
        // determine as intersecoes e a visibilidade
        // da linha
  }
  if (visibilidade == TRUE){
        // desenhe a linha
  }
}

Algoritmo de Cohen-Sutherland

  • Para linhas parcialmente visveis, as intersees com a regio de recorte pode ser determinadas diretamente da equao da reta.
    • interseo esquerda: $y = m(x_L - x_1) + y_1$
    • interseo direita: $y = m(x_R - x_1) + y_1$
    • interseo acima: $x = x_1 + (1/m)(y_T - y_1)$
    • interseo abaixo: $x = x_1 + (1/m)(y_B - y_1)$
    • $m=(y_2 - y_1)/(x_2 - x_1)$

Algoritmo de Cohen-Sutherland

  • Para cada aresta da janela (total de 4 arestas):
    • Para cada linha P1P2, determine se a linha totalmente visvel ou trivialmente invisvel.
    • Se P1 estiver fora da janela, continue, caso contrrio, troque P1 e P2.
    • Troque P1 pela interseo de P1P2 e a aresta da janela.

Algoritmo de Cohen-Sutherland

  • endcodes para P1 e P2: 0001 e 1000
  • aresta esquerda interceptada por P1:
    • $m = \frac{5 - (-2)}{3 - (-6)} = 7/9$
    • $y = 7/9 (-4 + 6) + (-2) = -4/9$
    • faa $P_1 (-4, -4/9)$
  • novos endcodes para P1 e P2: 0000 e 1000
  • aresta direita no interceptada
  • aresta inferior no interceptada
  • aresta superior interceptada por P2:
    • P1 no est do lado de fora da janela
    • Troque P1 por P2: $P_1(3,5)$ e $P_2 (-4,-4/9)$
    • $x = 3 + 9/7 (4-5) = 12/7$
    • faa $P_1(12/7, 4)$
  • novos endcodes para P1 e P2: 0000 e 0000. A linha totalmente visvel
  • trace a linha de $P_1(12/7, 4)$ a $P_2 (-4,-4/9)$!

Algoritmo de Cyrus-Beck

  • Objetivo: encontrar os limites do parmetro t na faixa [0,1] que determinam o segmento de reta recortado.
  • Para cada fronteira i,
    • $\hat{n_i} \cdot (P(t)-f_i) = 0$
    • $\hat{n_i} \cdot (P_1 + (P_2 - P_1)t-f_i) = 0$
    • $\hat{n_i} \cdot (P_1-f_i) + \hat{n_i} (P_2 - P_1)t = 0$
    • reescrevendo: $t (\hat{n_i} \cdot D) + w_i \cdot \hat{n_i} = 0$
  • O vetor $D = P_2 - P_1$ determina a direo da linha
  • O vetor $w = P_1 - f_i$ um fator de ponderao
  • O valor $t = - \frac{w_i \cdot \hat{n_i}}{D \cdot \hat{n_i}}$ define a posio do ponto na reta
  • Quando $D \cdot \hat{n_i} > 0$, o ponto encontra-se prximo ao incio do segmento
  • Quando $D \cdot \hat{n_i} < 0$, o ponto encontra-se prximo ao fim do segmento
  • Soluo: encontrar o maior valor de t prximo ao incio do segmento e o menor valor de t prximo ao final do segmento.
