Algoritmos de preenchimento

  • Servem para definir o conjunto de pixels que ser desenhando dentro de um determinado contorno fechado. Este contorno geralmente pode ser representado na forma poligonal.
  • A triagem dos pixels normalmente feita dentro de uma regio, denominada bounding box, ou caixa limitante, como mostra a figura.

Converso de varredura

  • Exceto nas bordas, pixels adjacentes em um polgono possuem as mesmas caractersticas. Esta propriedade chamada coerncia espacial. Assim, os pixels de uma dada linha (scan line) variam somente nas bordas do polgono.
  • O processo de determinar quais pixels sero desenhados no preenchimento chamado converso de varredura (scan conversion).

Algoritmo da lista ordenada de arestas

  • Tcnicas alternativas para preenchimento de polgonos utilizam a ordenao das interseces entre as arestas do polgonos e as scanlines (ordered edge list algorithm).
  • Algoritmo da lista ordenada de arestas
    • Determine para cada aresta as interseces com as $(y+1/2)$ scanlines, via Bresenham.
    • Armazene as interseces $(x, y+1/2)$ em uma lista.
    • Ordene a lista obtida da seguinte forma:
      • $(x_1, y_1)$ precede $(x_2, y_2)$ se $y_1 > y_2$ ou $y_1 = y_2$ e $x_1 \le x_2$.
    • Extraia os pares de elementos da lista, $(x_1, y_1)$ e $(x_2, y_2)$.
    • Ative os pixels da scanline $y$ para valores inteiros de $x$ tais que $x_1 \le x+1/2 \le x_2$.

Lista ordenada de arestas usando lista de arestas ativas

  • O novo algoritmo ainda necessita de muita memria alocada para armazenar as listas de interseces. Utilizar uma lista encadeada! Uma lista de fronteiras ativas indica para o algoritmo as arestas presentes (ativas) em cada scanline.
  • Algoritmo com lista de arestas ativas:
    • Preparao dos dados
      • Determine para cada aresta as interseces com as $(y+1/2)$ scanlines, via Bresenham, as maiores scanlines interceptadas pela aresta.
      • Armazene a aresta do polgono no y bucket da scanline correspondente.
      • Armazene a interseco inicial x, o nmero de scanlines interceptadas pela aresta, $\Delta y$, e o incremento de x, $\Delta x$, de scanline para scanline em uma lista encadeada.
    • Converso dos dados
      • Para cada scanline, verifique o aparecimento de novas arestas nos buckets correspondentes, e adicione a aresta lista de arestas ativas.
      • Ordene as interseces da lista de arestas ativas ($x_1$ precede $x_2$ se $x_1 \le x_2$).
      • Extraia os pares de elementos da lista, $(x_1, y_1)$ e $(x_2, y_2)$.
      • Ative os pixels da scanline $y$ para valores inteiros de $x$ tais que $x_1 \le x+1/2 \le x_2$.
      • Para cada aresta na lista de arestas ativas, decremente $\Delta y$ por 1. Se $\Delta y < 0$, remova aresta dessa lista.

Preenchimento baseado em semente

  • Algoritmos de preenchimento baseado em semente, ou seed fill algorithms, assumem que pelo menos um ponto no interior do polgono conhecido.
  • O algoritmo tenta encontrar o restante dos pontos no interior e preench-los com uma determinada cor.
  • necessrio informar o tipo de conectividade da regio. As regies podem ser 4-conectadas ou 8-conectadas.
  • Para uma regio 4-conectada, todos os pixels no seu interior podem ser alcanados com combinaes dos movimentos:
    • leste, oeste, norte e sul.
  • Para uma regio 8-conectada, os pontos no interior podem ser alcanadas com combinaes dos movimentos:
    • leste, oeste, norte, sul, nordeste, noroeste, sudoeste e sudeste
  • As regies 4-conectadas so delimitadas por fronteiras 8-conectadas e vice-versa.

Preenchimento baseado em semente

  • Algoritmo seed fill para regies 4-conectadas 
// seed(x,y)  o pixel semente
// push/pop: coloca/retira o pixel em uma pilha
// new_color: nova cor a ser preenchida
// boundary_color: cor do contorno
// pixel.left(): pixel on the left of current pixel
// pixel.right(): pixel on the right of current pixel
// pixel.top(): pixel above the current pixel
// pixel.bottom(): pixel below the current pixel
Pilha p;
Pixel pixel;
pixel.setPixel(seed(x,y));
p.push(pixel);
while(!p.empty()){
  pixel=p.top();
  p.pop();
  if(pixel.color != new_color){
        pixel.setColor(new_color);
        if(pixel.right().getColor() != new_color &&
           pixel.right().getColor() != boundary_color){
          p.push(pixel.right());
        }
        if(pixel.left().getColor() != new_color &&
           pixel.left().getColor() != boundary_color){
          p.push(pixel.left());
        }
        if(pixel.top().getColor() != new_color &&
           pixel.top().getColor() != boundary_color){
          p.push(pixel.top());
        }
        if(pixel.bottom().getColor() != new_color &&
           pixel.bottom().getColor() != boundary_color){
          p.push(pixel.bottom());
        }
}

Algoritmo scanline seed fill

  • Embora simples, o algoritmo anterior consome muita memria com o uso de pilhas.
  • A pilha pode conter freqentemente informao duplicada.
  • O algoritmo scanline seed fill contorna este problema semeando apenas um pixel nos trechos de uma scanline a ser preenchida.
    • Algoritmo scanline seed fill
      • Enquanto a pilha no estiver vazia…
      • Retire um pixel semente de um trecho de uma pilha contendo a semente.
      • Preencha os trechos esquerda e direita da semente, at que uma fronteira seja encontrada.
      • Grave as coordenadas da extrema esquerda (Xleft) e da extrema direita (Xright) do trecho preenchido.
      • Na faixa $Xleft \le x \le Xright$, para as scanlines imediatamente superior e imediatamente inferior, verifique se existem apenas pixels de fronteiras ou previamente preenchidos.
      • Se estas scanlines no contiverem apenas pixels de fronteiras ou previamente preenchidos, marque com uma semente o pixel da extrema direita de cada um dos trechos encontrados na faixa $Xleft \le x \le Xright$.

Funcionamento do algoritmo scanline seed fill

  • Os nmeros mostrados dentro dos pixels representam a posio da semente na pilha de sementes.
